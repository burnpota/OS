Red Hat Technical Account Manager 한진구 입니다.

여러개의 swap을 묶는 것과 관련되어 확인한 결과, 사용하시는데 있어서 특별한 장애 포인트는 발견하지 못하였습니다.


그리고 몇 가지 첨언을 드리면, 흔히 swap 사용율이 높은 시스템에서는 performance 향상을 위해서 별도의 빠른 디스크를 swap용으로 사용하는 경우가 있습니다.
거기에 더불어 swap을 마치 LVM stripe와 같은 기능으로 사용을 하여 performance를 향상하는데 도움이 되기도 합니다.


다만 보내주신 예제를 보면,


[root@localhost ~]# cat /etc/fstab
...
UUID=ee7d5bdb-53e6-4a15-8d36-f396cafbc5cf


swap
swap

defaults,pri=2

0 0
UUID=e7ae6cd8-6922-4ea4-b91c-596ec1e4f4a6


swap
swap

defaults,pri=1

0 0


위에 pri={1,2} 값이 설정된 것이 보입니다.

이 pri= 값에 대해서 동일한 값으로 설정하게 될 경우, round-robin으로 swap 발생시 사용하게 되는 형태로 구성해보는 것에 대해서 검토해보실 것을 제안드립니다.
