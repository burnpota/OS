안녕하세요,

해당 이슈에 대하여 확인해본바에 의하면 아래와 같습니다.

"Timer popped" 메시지는 아래와 같은 함수에 의하여 출력됨

gboolean
action_timer_callback(gpointer data)
{
crm_warn("Timer popped (timeout=%d, abort_level=%d, complete=%s)",
timer->timeout,
transition_graph->abort_priority, transition_graph->complete ? "true" : "false");
}

해당 함수를 사용하는 함수는 아래와 같으며 이때 action의 timeout에 graph->network_delay 값을 추가하는것으로 확인됨

action->timer->source_id = g_timeout_add(action->timer->timeout + graph->network_delay,
action_timer_callback, (void *)action->timer);

graph->network_delay 값은 cluster-delay 값에 의하여 결정 되는것으로 확인

time = crm_element_value(xml_graph, "cluster-delay");
CRM_CHECK(time != NULL, free(new_graph);
return NULL);
new_graph->network_delay = crm_get_msec(time); <=========

cluter-delay 의 디폴트 값은 60초입니다.

cluster-delay = time [60s]
Round trip delay over the network (excluding action execution)

따라서 해당 이슈에서 action의 timeout 값을 120초로 설정시 실제 timeout에 걸리는 시간은 180초로 됩니다.

감사합니다.
