안녕하세요.

RHEL7.1 -> RHEL7.2 로 업그레이드 후 /etc/profile.d/PackageKit.sh 파일이 변경되어 삼성생명 내 보안 프로그램에서 제대로 스크립트를 해석 못하는 문제가 발견되었습니다.

보안 프로그램이 아예 실행이 되지 않거나 로그인에 문제가 되지는 않지만, 초기 Login 시 Parse 에러가 지속적으로 발생하고 있습니다.

문제를 찾아보니 /etc/profile.d/PackageKit.sh 의 14line [[ $- =~ i ]] || runcnf=0 부분에서 파싱을 못하는 것 같습니다.

이와 관련하여 보안솔루션 업체에도 동일한 문의를 진행 중이나, 궁금한 내용은 PackageKit 의 역할 입니다.

해당 스크립트가 PackageKit-command-not-found 패키지에 같이 포함되어 있는데, 이 프로그램의 정확한 역할을 알고 싶습니다.

Red Hat Technical Account Manager 한진구 입니다.

'PackageKit-command-not-found' 패키지는 bash 쉘에서 어떤 특정 command를 실행하려고 할때, 실행 명령어가 설치되어 잇지 않은 경우, 설치할 수 있도록 도와주는 패키지입니다.

예를들어, 'powertop'이란 명령어를 쳤을때, 시스템에 설치가되어있지 않은 경우에 이를 쉽게 설치할 수 있도록 아래와 같이 설치를 도와줍니다.

> Command not found. Install package 'powertop' to provide command 'powertop'? [N/y]
