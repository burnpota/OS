======================
사전문의

어떤 문제/오류/결함이 발생했습니까? 기대하시는 결과는 무엇입니까?

dmz 에 위치한 서버들과 내부망에 위치한 ntp 장비간의 정상통신 점검이 필요합니다.
#telnet IP port 와 같은 방식으로 점검을 해보고 싶어서 udp는 어떻게 점검이 가능한지
검색을 해보니 , nc 라는 커맨드를 사용하면 된다고 해서 아래와 같이 이용해봤습니다.

root@SLPCHTWL0B /root # nc -v -u 100.254.147.100 123
Ncat: Version 6.40 ( http://nmap.org/ncat )
Ncat: Connected to 100.254.147.100:123.
^C
root@SLPCHTWL0B /root # nc -v -u 100.254.147.100 124
Ncat: Version 6.40 ( http://nmap.org/ncat )
Ncat: Connected to 100.254.147.100:124.
^C
root@SLPCHTWL0B /root # nc -v -u 100.254.147.100 1000
Ncat: Version 6.40 ( http://nmap.org/ncat )
Ncat: Connected to 100.254.147.100:1000.
^C
root@SLPCHTWL0B /root # nc -v -u 100.254.147.99 123
Ncat: Version 6.40 ( http://nmap.org/ncat )
Ncat: Connected to 100.254.147.99:123.
^C

보여드린 것처럼 아무 IP나 포트나 넣어도 전부 connected 가 나오는걸 발견했습니다.

뭔가 옵션이 잘못된건지, 확인 부탁드립니다.

현재 방화벽 담당자와 확인해보니, 해당 장비와는 방화벽에서 deny 상태가 맞다고 합니다. ^^

실제 deny 되고 있는데 nc -v -u IP 포트를 주면 정상 connected 메시지가 나오고 있는 상태라,
udp 접속테스트를 하는 방법 확인 부탁드립니다.
