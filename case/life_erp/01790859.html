그리고 지금 rear local.conf화일을 multiple로 사용하고자 합니다.
regular는 클론으로 스케쥴링하여 구동 예정이고
irregular는 작업시 임시적으로 백업하는 설정입니다.
아래와 같은 경로의 파일로 구동할 예정이며
regular쪽은 rear -v -d mkbackup명령어로
irregular쪽은 rear -c /etc/rear/irregular/ mkbackup -v -d
명령어로 실행하고자 합니다.
이런 설정에 문제가 있는지 확인 부탁드리고 더 추가하면 좋을 사항에 대해서 가이드 부탁드립니다.
======================

/etc/rear/local.conf
OUTPUT=ISO
OUTPUT_URL=nfs://x.x.x.x/nfs_data/regular
BACKUP=NETFS
BACKUP_URL=nfs://x.x.x.x/nfs_data/regular
BACKUP_PROG_ARCHIVE="$(date +%y%m%d)_$(hostname)_regular"
BACKUP_PROG_EXCLUDE=("${BACKUP_PROG_EXCLUDE[@]}" '/media' '/var/tmp' '/var/crash' '/CRASH' '/dev/' '/sys' '/proc' '/mnt' '/tmp' '/run' '/media' '/lost+found' '/osbackup')

⁠Chapter 27. System and Subscription Management

..........
⁠ ReaR works only on the eth0 interface
ReaR produces a rescue system that does not support mounting an NFS server using an interface other than eth0. Consequently, the backup files cannot be downloaded and the system cannot be restored. To work around this problem, ensure that the used interface is eth0 by restarting dhclient. (BZ#1313417)


위의 의미는 ReaR 에 의해서 제작된 rescue 시스템이 eth0 가 아닌 다른 디바이스를 사용해서 NFS 서버로 마운트하는것을 지원하지 않는다
는 의미입니다. 이 이슈는 아래의 버그질라[1]에서 보고가 되어졌고, 인터페이스가 여러개인 시스템에서 eth0 가 아닌 eth1 에 IP 가 할당
이 되었다가 NFS 서버로 마운트가 실패되는 이슈로 보고가 되었다가 이는 버그가 아니라고 판정을 하였고 이를 해소하기 위해서는 dhclient
를 재시작하여 eth0 에 IP 가 할당이 되도록 한 뒤에 NFS 서버로부터 마운트하여 복구가 가능하다는 내용입니다.
