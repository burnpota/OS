안녕하세요? 레드햇 허경입니다.

1. 해당 서버가 /etc/sysconfig/clock으로 설정이 되어있으면 edt 시간으로 맞춰져서 보이게 되는 것인가요?
(시간은 한국 시간이지만 date의 결과만 edt로 보이게 되는 것인가요?)
TZ="Asia/Seoul" date 명령과 같이 변수를 적용하시면 원하신 시간대로 시간을 표시할 수 있습니다.

2. /etc/sysconfig/clock 부분이 처음에 redhat os 설치할 때 지도에서 선택하는 나라로 되는 것인지 문의드립니다.
설치시 선택하는 도시입니다.

3. 만약 한국에 있는 서버인데 설정을 잘못하여서 America/New_York으로 되어있을 경우 다시 KST로 표현하는 방법 관련해서 문의드립니다.

아래와 같이 변경 가능합니다.

변경 전
# The time zone of the system is defined by the contents of /etc/localtime.
# This file is only for evaluation by system-config-date, do not rely on its
# contents elsewhere.
ZONE="America/New_York"
[root@rhel6u4-1 ~]# date
2018. 05. 08. (화) 13:49:22 EDT

설정 변경
[root@rhel6u4-1 ~]# rpm -ql tzdata | grep Seoul
/usr/share/zoneinfo/Asia/Seoul
/usr/share/zoneinfo/posix/Asia/Seoul
/usr/share/zoneinfo/right/Asia/Seoul

[root@rhel6u4-1 ~]# cp /usr/share/zoneinfo/Asia/Seoul /etc/localtime

[root@rhel6u4-1 ~]# md5sum /etc/localtime
0a36916e2e979dbcfd37c789955f1aba /etc/localtime

[root@rhel6u4-1 ~]# strings /etc/localtime
TZif2
Dp!n=`"
&p#N
TZif2
!n=`
KST-9

[root@rhel6u4-1 ~]# md5sum /usr/share/zoneinfo/Asia/Seoul
0a36916e2e979dbcfd37c789955f1aba /usr/share/zoneinfo/Asia/Seoul

[root@rhel6u4-1 ~]# cat /etc/sysconfig/clock
# The time zone of the system is defined by the contents of /etc/localtime.
# This file is only for evaluation by system-config-date, do not rely on its
# contents elsewhere.
ZONE="Asia/Seoul"

변경 후
[root@rhel6u4-1 ~]# date
2018. 05. 09. (수) 02:57:23 KST
