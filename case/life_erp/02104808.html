안녕하세요, Red Hat Technical Account Manager 허경입니다. 지난 05월 22일 공개된 관련 안내입니다.

시스템에서 권한이 없는 공격자는 Speculative Store Bypass 보안 취약점을 사용하여 일반적인 메모리 보안 제한을 우회하여 접근 권한을 필요로 하는 메모리에 대한 읽기 권한을 획득할 수 있습니다. 보안 취약점 문제는 CVE-2018-3639로 지정되어 있으며 "변형 (Variant) 공격 4 " 또는 "추측성 저장 우회 (Speculative Store Bypass)" 표기하고 있습니다.

자세한 보안 취약점 설명에 대해서는 아래 페이지를 방문후 배경정보(Background Information) 확인을 부탁드립니다.

https://access.redhat.com/ko/security/vulnerabilities/3453341(한글)
https://access.redhat.com/security/vulnerabilities/ssbd (영문)

* CVE버젼 :
- https://access.redhat.com/security/cve/cve-2018-3639(Important)

* 영 향 도
1) 심각도 등급 : 중요 (Important)

2) 영향 조건
- 영향에 관련 패키지 업데이트
- https://access.redhat.com/security/vulnerabilities/ssbd 에 방문 후, Diagnose(진단)에 있는 detection script(진단 스크립트)를 사용하여 시스템에서 확인이 가능합니다.

3) 각 버젼 별 패키지 목록

- 관련 패키지 목록에 대해서는 'Resolve' 탭을 참조 해주시길 바랍니다.
https://access.redhat.com/security/vulnerabilities/ssbd

해당 패치는 시스템의 Downtime을 요구하는 작업이며, 해당 보안 취약점은 로컬에 인가되지 않는 공격자에 의한 보안취약점이며, 서비스에 영향이 없도록 업데이트 일정을 계획하시어 해당 패치 버전으로 업데이트를 권장 드립니다.
