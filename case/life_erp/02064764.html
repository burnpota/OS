안녕하세요

Red Hat Global Support Services 를 이용해 주셔서 감사합니다.

3개까지의 커널 버전이 유지되며, 만일 이전 커널이 업데이트로 인하여 삭제된 경우 원복을 진행하려면, 두 가지 방법이 존재합니다.

1. yum history 를 이용하여 원복
- 이 방법은 yum 작업시에 트랜잭션을 저장해두고 이전 환경으로 롤백하며, 서브스크립션으로 등록된 시스템에 한하여 적용이 가능합니다.
# yum history
# yum history undo 번호

2. 수동으로 패키지 원복

- 3개의 커널 중에 남겨두어야할 커널로 부팅한 이후에 삭제가 필요한 커널을 삭제한 이후, 이전 버전의 커널을 설치하면됩니다.

- 예를 들어 현재 설치된 커널이 아래와 같고, 2.6.32-504 로 롤백을 해야하는 경우라면, 3개의 커널 중에 삭제를 진행한 이후에 이전 버전의 커널을 설치해야합니다.
kernel-2.6.32-504.16.2.el6
kernel-2.6.32-504.12.2.el6
kernel-2.6.32-504.8.1.el6

- 혹은 삭제를 진행하지 않고 새로운 커널을 설치하는 경우 오래된 커널이 자동으로 선택되어 삭제가 진행된 후 설치가 진행됩니다.

- 2.6.32-504.el6 로 원복하려는 경우 현재 커널이 삭제할 커널이라면 리부팅 이후 다른 커널로 부팅한 이후 삭제해야할 커널을 remove 합니다.
# uname -a
kernel-2.6.32-504.16.2.el6

# reboot

# uname -a
kernel-2.6.32-504.12.2.el6

# yum remove kernel-2.6.32-504.16.2.el6.x86_64

- 삭제 후 필요한 커널을 설치합니다.
# yum install kernel-2.6.32-504.el6.x86_64.rpm

의존성 관련 패키지는 수동으로 찾아서 삭제/설치를 진행해야합니다.

안녕하세요? 레드햇 허경입니다.

kernel-firmware 패키지의 경우 시스템에 여러개의 버전이 동시에 존재할 수 없습니다.
kernel 패키지는 여러 개의 버전이 시스템에 공존할 수 있습니다.
이런 이유로 kernel-firmware는 update를 하고 kernel은 install을 하는 경우가 많습니다.

yum update 명령의 경우 kernel-firmware와 같이 여러개의 버전이 시스템에 동시에 설치될 수 없는 패키지는 업데이트가 이루어집니다.
하지만, kernel 패키지의 경우 여러 개의 버전이 공존이 가능하기 때문에 yum update 명령을 수행해도 이전 패키지가 지워지지 않고 남아있게 됩니다.

아래 내용은 제가 테스트 환경에서 수행한 yum update 명령 결과입니다.

# yum update kernel-2.6.32-504.16.2.el6 kernel-firmware-2.6.32-504.16.2.el6

yum update 명령이지만, kernel 패키지는 install이 되고 kernel-firmware 는 업데이트가 진행됩니다.
저의 테스트 환경의 경우 kernel-firmware의 최신 버전이 존재하기 때문에 yum update 명령에서 지정한 버전보다 최신 버전으로 업데이트 됩니다.

구성하신 update용 repo를 이용하는 환경이라면 kernel-firmware 패키지는 repo에 존재하는 버전으로 업데이트가 될 것입니다.
따라서, 지금 현재 구성하신 환경에서는 yum update kernel-firmware를 수행한 후 yum install kernel을 수행하시는 것과 yum update를 한번에 수행하시는것이 동일한 결과를 가져올 것으로 예상됩니다.

kernel을 제외한 패키지들은 모두 업데이트가 이루어지고 kernel만 설치가 된다고 생각하시면 됩니다.
