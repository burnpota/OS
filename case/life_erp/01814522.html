제목 : How to backup and recover RHEL where is in SAN boot disk with mpath using ReaR? [ Rear백업 사용시 OS영역이 SAN스토리지 환경에서의 OS백업/복구 방법 ]

======================
사전문의

Rear백업 복구시에 OS환경이 SAN 스토리지 영역에 존재할 경우 OS역역 백업복구 방법에 대해서 가이드 부탁드립니다

해당 파티션이 SAN 환경 즉 OS영역이 스토리지에 존재하는 환경입니다
해당 환경에서 OS를 백업 / 복구를 진행이 가능하진 가능하다면 어떠한 방법으로 Rear 백업/복구가 가능한지 확인 부탁드립니다
참고적으로 OS부분은 멀티패스로 구성되어 있습니다
자세한 내용으로 sosreport파일 첨부드립니다

안녕하세요,

Red Hat Global Support Service 를 이용해주셔서 감사합니다.

vi /usr/rear/rear.conf
~~ 생략
# Automatically exclude multipath disks and their dependent components
AUTOEXCLUDE_MULTIPATH=N
~~ 생략
# to force adding multipath related executables so a recovered system would be able to boot via SAN disks only
# instead of the internal ones (no success guaranteed although). E.g. in case the destination has no internal disks.
# making the variable (y,Y,1) to enable
BOOT_OVER_SAN=Y
~~ 생략

저희 전문그룹으로부터 답변이 오지 않아 자체적으로 확인을 해보니 OS 가 SAN 환경으로부터 부팅이 필요할때 환경일때 쓰이는
두가지 옵션을 찾았습니다. 이 옵션들을 위와 같이 설정한 뒤에 backup & recovery 를 시도해보시고 피드백을 해주시면
계속해서 지원을 하도록 하겠습니다.


[ 참고문서 ]

[1] What is Relax and Recover(ReaR) and how can I use it for disaster recovery?
- https://access.redhat.com/solutions/2115051

감사합니다.

ReaR 의 upstream 프로젝트에서도 여전히 논의를 하고 있는것으로 확인을 하였고 이것이 공식적으로 지원이
