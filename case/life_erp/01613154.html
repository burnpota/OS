'open-vm-tools'의 debugging 모드를 활성화 하는 방법을 안내드립니다.

# vi /etc/vmware-tools/tools.conf

[logging]

log = true
vmtoolsd.level = debug
vmtoolsd.handler = file
vmtoolsd.data = /var/log/vmtoolsd.log


위와같이 설정파일을 수정한 후에, 'vmtoolsd'서비스를 재 가동하여 주시기 바랍니다.

@ open-vm-tools 이 무엇인지?

The open-vm-tools package is a suite of open source virtualization utilities and drivers which improve performance and user experience when running Red Hat Enterprise Linux 7 as a guest on a VMware vSphere ESX(i) hypervisor. The open-vm-tools package provides a secure and authenticated mechanism for performing various operations on the guest operating system from the hypervisor.

(https://access.redhat.com/articles/1282083)


@ open-vm-tools가 무엇을 하는지?

The open-vm-tools package contains the core open-vm-tools userspace programs and libraries, including the vmtoolsd service, which controls the functions in the open-vm-tools package. The vmtoolsd service starts by default when the guest operating system boots and passes information between host and guest.

The open-vm-tools package provides the capability for:

File transfer between a host and guest
Improved virtualized memory management and network performance
Mechanisms and protocols for communication between host and guests
Synchronizing the guest operating system clock with the VMware host clock
Infrastructure for performing graceful power operations (shut down) and file system quiescing of virtual machines
Running scripts to automate guest operating system operations
Accepts additional plug-ins that can extend or customize open-vm-tools functionality

(https://access.redhat.com/articles/1282083)
