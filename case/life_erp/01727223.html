안녕하세요,
Red Hat 한진구 입니다.

시스템 리부팅 시점이 어려움에 따라 Runtime으로 해당 패치를 임시적으로 반영 할 수 있는 방법을 문의 주셔서 답변 드립니다.

우선 두가지 방법이 있으며, 아래 두가지 방법에 대해서 설명드리오니 검토 후 결정사항을 안내해주시면 지원하도록 하겠습니다.


1. kpatch (실시간 커널 패치를 반영하는 방안)

Kpatch는 runtime시에 버그 패치를 반영하는 것으로 제약사항은 아래와 같습니다.

우선 RHEL7.2 부터 Red Hat은 지원을 하고 있는 상태이며, 현재 사용중인 kernel의 버젼이 RHEL7.2 GA상태의 버젼이어야 합니다. (EUS는 지원되지 않음)
kernel ver. kernel-3.10.0-327.el7


그리고 kpatch는 성능저하에 영향을 줄 수 있다는 단점 또한 존재하기 때문에 이 점은 참고하시기 바랍니다.
======================
참고로 RHEL5 와 RHEL6는 적용 대상이 되지 못합니다.

관련 링크는 아래와 첨부드리며 만약 kpatch를 결정하시게 되시면 안내주시면 Request를 올려서 kpatch를 받을 수 있도록 진행하도록 하겠습니다.

KCS Document, https://access.redhat.com/solutions/2206511
======================
2. SystemTap

SystemTap은 기본적으로 커널을 디버깅할때 사용하는 방식입니다. 그러다보니 시스템의 성능에 영향을 많이 주는 방식이기는 하나 버젼에 큰 제약은 없어 전반적으로 모든 시스템에 통일되게 운영하기에는 장점이 있는 방식이 될 것같습니다.

SystemTap 안내에 대한 자료는 아래 문서를 참고로 포함해드립니다.

KCS Document, https://access.redhat.com/articles/882463
======================
적용은 아래와 같이 작업하시면 됩니다.

1) On the host, save the following in a file with the ".stp" extension:

probe kernel.function("mem_write").call ? {
$count = 0
}

probe syscall.ptrace { // includes compat ptrace as well
$request = 0xfff
}

probe begin {
printk(0, "CVE-2016-5195 mitigation loaded")
}


probe end {
printk(0, "CVE-2016-5195 mitigation unloaded")
}


2) Install the "systemtap" package and any required dependencies. Refer
to the "2. Using SystemTap" chapter in the Red Hat Enterprise Linux
"SystemTap Beginners Guide" document, available from docs.redhat.com,
for information on installing the required -debuginfo and matching kernel-devel packages

3) Run the "stap -g [filename-from-step-1].stp" command as root.

If the host is rebooted, the changes will be lost and the script must be
run again.

안녕하세요,
Red Hat Red Hat Technical Account Manager 한진구 입니다.

10월 20일 공개된 CVE-2016-5195 관련 안내입니다.

리눅스 시스템에서 공격자가 리눅스 커널의 Read-only 영역 메모리에 쓰기 권한을 줄 수 있는 취약점이 발견되었으며, 또한 루트 권한을 획득 역시 가능한 보안이슈 입니다.
따라서, 현재 언론에서도 보도가 되고 있는 보안이슈로써, 신속하게 대응하실 수 있도록 사전 안내드립니다.


* CVE 버전 : https://access.redhat.com/security/cve/cve-2016-5195
- 대상 버전 : Red Hat Enterprise Linux 4,5,6,7 및 파생 버젼
- 관련 패키지 : kernel
- 취약점 내용 :


이 보안 취약점은 읽기전용 메모리 맵핑의 COW(Copy-On-Write) 결함를 다루는 리눅스 커널의 메모리 하위시스템 관련된 것으로,
공격자는 이 결함을 통해 읽기전용 메모리의 쓰기 권한을 획득 하면서 root 권한으로 실행되는 프로세스의 중간에 개입하여 루트의 권한을 획득 가능한 보안이슈입니다.
======================
