제목 : 리눅스에서 /etc/resolv.conf DNS 설정 시 캐슁 여부 문의

======================
사전문의

어떤 문제/오류/결함이 발생했습니까? 기대하시는 결과는 무엇입니까?

안녕하세요.

SAP AP 시스템 배포시 사용되는 전체 노드의 호스트 수량이 많아 /etc/hosts 대신 /etc/resolv.conf 의 DNS 를 설정하여 구축하려고 계획 중입니다.

MS Windows 경우 자체 DNS 쿼리 내역에 대한 캐슁기능이 있어 매번 재쿼리 하는 부하가 없는 것으로 알고 있는데,

RHEL 의 경우 기본적으로 이와 유사한 캐슁기능을 제공하는지 궁금합니다.

알아본 결과 nscd 데몬이 비슷한 기능을 제공하는데, 캐슁 기능을 위해 꼭 사용해야 하는지 문의 드립니다.

안녕하세요.

질문드리는 시스템은 DNS 데몬 운영서버가 아닌 일반 AP 서버 입니다.

SAP 구축시 사용되는 호스트가 많아 이를 /etc/hosts 가 아닌 DNS 에 등록하여 모든 노드에서 DNS 쿼리를 하려는 목적입니다.

매번 Domain 주소 검색 시 DNS 에 쿼리를 한다면 이것도 작지만 부하이기에,

도메인 쿼리를 하는 클라이언트 측 에서의 캐슁 기능을 문의 드린 것입니다.

RHEL 리눅스 자체에서 기본적으로 접속했던 도메인에 대해 캐쉬를 하는지 여부에 대해 다시 한번 문의 드립니다.
