안녕하세요.

2017-10-21 02:50 경 Kernel 에서 알 수 없는 panic 이 발생하여 vmcore 가 생성되었습니다.

해당 서버의 용도는 Backup 이며, 시스템에 NetBackup 이 설치되어 있는 시스템 입니다.

어디서 문제가 발생했습니까? 어떤 환경에서 발생했습니까?

이전에 DBKMML01SL 시스템에서 2017-09-24 11:22:10 경에도 알수없는 panic 이 발생하여 vmcore 가 생성된 적이 있는데요,

당시 이슈는 이미 알려진 https://access.redhat.com/solutions/2647381 (Kernel panic or soft lockup in netlink_compare) 이슈였습니다.

당시 생성했던 케이스는 아래와 같습니다.

https://access.redhat.com/support/cases/#/case/01938341

언제 문제가 발생했습니까? 이러한 문제가 자주 발생합니까? 반복적으로 발생합니까? 특정 시간에 발생합니까?

이번에도 발생했던 시각 SAR 데이터등을 확인해 봐도 특이사항은 발견할 수 없었으며,

/var/log/messages 파일에도 사전 이상증상을 발견할 수 없었습니다.

다행히 vmcore 가 생성되어 kernel panic 이라는 것을 유추할 수 있었습니다.

이번 장애도 지난 번과 동일한 이슈인지 확인이 필요할 것 같습니다.

PBKSML02SL 도 DBKMML01SL 마찬가지로 NetBackup 을 사용하는 백업용도의 서버 입니다.

해결 기간이나 업무에 미치는 영향에 대한 정보를 제공해 주시겠습니까?

관련 분석 데이터를 아래와 같은 파일명으로 dropbox.redhat.com/incoming 에 업로드 하였습니다.

안녕하세요

Red Hat Global Support Services 를 이용해 주셔서 감사합니다.

ghes.disable=1 의 경우 APEI Generic Hardware Error Source 규격에 맞지않는 정보가
전달되었을때, Panic 대신에 처리 자체를 비활성화 한다는 의미입니다.


if (ghes_disable) {


pr_info(GHES_PFX "GHES is not enabled!\n");


return -EINVAL;

}

문제는 실제로 오류가 있는 장치 감지가 어려울 가능성도 있지만, false positive(오보) 일 가능성도 있다는 것입니다.
하드웨어 오류가 아닌 부분이라면 해당 기능을 비활성화하여도 시스템 운영에는 지장이 없을 것으로 보입니다.

만약 본 이슈와 관련하여 추가적인 문의 사항이 있으실 경우 연락 주시기 바랍니다.
