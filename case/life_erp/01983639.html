안녕하세요,

Red Hat Global Support Service를 이용해주셔서 감사합니다.

케이스 이력을 확인해보니
이전 01967001 를 통해 저희 기술지원 엔지니어로부터 동일 질문에 대해서 안내 받으신 것으로 확인됩니다.
해당 TCPBacklogDrop 은 소켓 버퍼와 관련있으며
첨언 드리면

데이터를 전송하는 TCP 세그먼트가 도달할 때 데이터 페이로드는 소켓 버퍼 데이터 영역에 놓여집니다.
그러나 만약 소켓 버퍼가 현재 유저 스페이스에 의해 락킹 된다면
동시에 커널은 소켓 버퍼 영역에 접근할 수 없게 됩니다.

그러한 경우에, 패킷은 socket receive backlog 에 놓여지며
소켓 버퍼 데이터 영역을 차지하게 됩니다.

만약 소켓 버퍼 데이터 영역이 해당 시점에 꽉 차게 되면 더이상 패킷을 보관할 공간이 없어지고 패킷은 drop 됩니다.
이 때, TCPBacklogDrop 가 증가합니다.

이것은 다른 소켓 버퍼 패킷 유실 처럼 처리될 수 있습니다:
개선책으로는 소켓 버퍼를 늘리시거나 어플리케이션의 소켓 리시브 퍼포먼스를 개선해보시기 바랍니다.

감사합니다.
