안녕하세요.

금번 삼성생명 보험ERP DR 구축 시 설정 값 등을 모두 적용한 kickstart 배포를 계획 중인데요,

KS 파일내 Partition 부분에 swap size 를 128G 이상으로 설정하면

disk sufficiently 메세지가 나오면서 에러가 발생합니다.

(SAP 의 Swap 권고 사항을 기준으로 설치를 하기 때문에 대용량의 swap 공간이 필요합니다.)

혹시 RHEL7 Install 시 max swap size 에 대한 제약 사항이 있는지 확인 부탁 드립니다.

어디서 문제가 발생했습니까? 어떤 환경에서 발생했습니까?

KS 내용 중 partition 부분은 아래와 같습니다.

# Partition clearing information
# clearpart --none --initlabel
clearpart --all --initlabel

# Disk partitioning information
part /boot --fstype="xfs" --asprimary --size=1024
part / --fstype="xfs" --asprimary --size=30720
part /var --fstype="xfs" --asprimary --size=20480
part swap --fstype="swap" --size=196608
part pv.01 --size 1 --grow

volgroup vg0 pv.01
logvol /home --vgname=vg0 --fstype="xfs" --size=10240 --name=home
logvol /sysadmin --vgname=vg0 --fstype="xfs" --size=20480 --name=sysadmin
