제목 : KDUMP 생성관련 /etc/sysconfig/kdump 파일 값 문의

======================
사전문의

안녕하세요

SDS 삼성생명 인프라 서비스 담당하고 있는 김지민 선임입니다.

아래와 같이 2대의 서버의 kdump kernel 부분이 있는데요.

두 대의 차이점은 nr_cpus=1 부분과 nr_cpus=6 disable_cpu_apicid=0 부분입니다.

redhat 7.2 release note에서 기술프리뷰 19장에 보면

AMD64 및 Intel 64 시스템 상의 kdump에서 여러 CPU 지원 부분이 있는데요.
해당 내용으로 해석하면

nr_cpus=1 설정이면 dump 생성 시 1개의 cpu만 이용하게되고
nr_cpus=6 disable_cpu_apicid=0 설정이면 dump 생성 시 6개의 cpu만 이용하는 것으로 해석을 하면 되는지 문의드리려고 합니다.


DEPSCL01SL = /etc/sysconfig/kdump
KDUMP_COMMANDLINE_APPEND="irqpoll nr_cpus=1 reset_devices cgroup_disable=memory mce=off numa=off udev.children-max=2 panic=10 rootflags=nofail acpi_no_memhotplug transparent_hugepage=never rd.driver.blacklist=RedCastle"

DECCAL01SL = /etc/sysconfig/kdump
KDUMP_COMMANDLINE_APPEND="irqpoll nr_cpus=6 disable_cpu_apicid=0 reset_devices cgroup_disable=memory mce=off numa=off udev.children-max=2 panic=10 rootflags=nofail acpi_no_memhotplug transparent_hugepage=never rd.driver.blacklist=RedCastle"
