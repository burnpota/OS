안녕하세요,
Red Hat Technical Account Manager 한진구 입니다.

우선 compat-libstdc++-33 패키지는 아시다시피 오래된 GCC 3.3.4 버젼의 라이브러리를 사용하는 application의 호환성을 위해서 제공하는 패키지 입니다.


> RHEL6버전의 iso에서는 해당 package가 있었었는데 RHEL7 버전에서는 혹시 포함되지 않은 이유가 있는 것인지 문의드리려고 합니다.

라이브러리도 계속 업데이트되고, RHEL도 버젼이 업데이트 되면서 오래되는 정도에 따라 지원에 대한 부분을 결정하게 되는데, 해당 compat-libstdc++-33 패키지는 RHEL6에서는 공식적으로 제공하였던 것이고, RHEL7에서는 공식적인 지원에서는 배제를 하나, 필요에 의한 고객들을 위해서 optional repository로 별도로 제공하고 있다고 생각하시면 될 것 같습니다.

(참고, https://access.redhat.com/solutions/300513 )

> 추가로 iso에 포함되지 않은 package 설치 시 해당 package로 인한 문제가 발생시
> redhat의 지원이 가능한지에 대해서도 문의드리려고 합니다.

해당 패키지에 대한 문제 발생시에 Red Hat 공식 지원은 불가능 합니다.


추가적으로, 첨언을 드린다면 해당 라이브러리를 사용하는 application도 새롭게 업그레이드가 되지 않앗을까 싶습니다. 이러한 부분도 application과 상의하시어 업그레이드 하는것도 향후를 위해서 필요해 보입니다.
======================
