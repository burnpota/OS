SDS 김지민 선임입니다.

사용하지 않는 서비스와 관련된 패키지 삭제 관련해서 문의드리려고 합니다.

대상은 삼성생명 ERP AP 서버 ECC 업무 1번 서버를 대상으로 확인하였고
동일하게 다른 쪽도 패키지는 설치되어있습니다.

아래는 삭제가 가능한지 문의하려는 패키지이고
systemctl status 등으로 서비스를 확인하였을 때 현재 OS에서는 사용하지 않는 것 처럼보이는 것은 표기를 하였습니다.

문의드리는 이유는 ERP 서버 보안 점검 관련해서 /var/log 밑의 파일들에 대해서 점검을 하였는데요. 아래 패키지들과 관련된 log 파일들에 대해서 소유자나 권한등의 문의가 와서 사용하지 않는 것이 확인되면 굳이 보안 기준에 맞춰서 권한을 바꿔 놓기보다는 삭제하는 것이 추후 관리하는데 편할 것 같아서 문의드립니다.

Red Hat Global Support Services를 이용해주셔서 감사합니다.

문의 주신 내용에 대해 아래와 같이 답변 드리겠습니다.
문의 주신 패키지들은 관련된 서비스들을 사용하시지 않으시다면 삭제하셔도 무방합니다.

tuned-2.5.1-4.el7.noarch 시스템 튜닝 서비스(mask됨), 사용하시지 않는다면 삭제 가능.
glusterfs-3.7.1-16.el7.x86_64 gluster storage를 사용하는 환경이 아니라면 삭제 가능.
chrony-2.1.1-1.el7.x86_64 chronyd 서비스 (NTP) 를 사용하시지 않는다면 삭제 가능.
389-ds-base-1.3.4.0-19.el7.x86_64 LDAP 미사용시 삭제 가능.
cups-filters-1.0.35-21.el7.x86_64 해당 서버로부터 프린터를 사용하시지 않는다면 삭제 가능.
gdm-3.14.2-12.el7.x86_64 GUI를 사용하는 시스템이 아니라면 삭제 가능.
qemu-guest-agent-2.3.0-4.el7.x86_64 vm이 아닌 환경이라면 삭제 가능.

패키지 정보를 얻으시려면 # yum info <패키지> 커멘드를 사용하실수 있습니다.
