공유해주신 정보를 기반으로 정리 하면 "JDK 1.5기반으로 WebSphere 실행 중 Java 소스 코드에 하드 코딩된 한글이 깨지는 현상" 입니다.
나머지 상세 정보는 정상적으로 제공되지 못 하고 있습니다.

Java 애플리케이션 실행 중 하드 코딩된 한글이 깨지는 이슈는
1. 소스 코드 파일 인코딩,
2. 컴파일 할 때의 Java compiler 인코딩,
3. JVM 실행 인코딩,
4. Character 인코딩
중 일치하지 않는 부분이 있기 때문입니다.

[참고]
* Java compiler 인코딩: javac -encoding <encoding>
* JVM 실행 인코딩: -Dfile.encoding=UTF-8
JBoss EAP JVM 예:
~~~
# jps -lmv
47814 /root/runtime/eap/jboss-eap-7.1/jboss-modules.jar -mp /root/runtime/eap/jboss-eap-7.1/modules org.jboss.as.standalone -Djboss.home.dir=/root/runtime/eap/jboss-eap-7.1 -Djboss.server.base.dir=/root/runtime/eap/jboss-eap-7.1/standalone -D[Standalone] -verbose:gc -Xloggc:/root/runtime/eap/jboss-eap-7.1/standalone/log/gc.log -XX:+PrintGCDetails -XX:+PrintGCDateStamps -XX:+UseGCLogFileRotation -XX:NumberOfGCLogFiles=5 -XX:GCLogFileSize=3M -XX:-TraceClassUnloading -Xms512m -Xmx512m -XX:MetaspaceSize=96M -XX:MaxMetaspaceSize=256m -Djava.net.preferIPv4Stack=true -Djboss.modules.system.pkgs=org.jboss.byteman -Djava.awt.headless=true -Dorg.jboss.boot.log.file=/root/runtime/eap/jboss-eap-7.1/standalone/log/server.log -Dlogging.configuration=file:/root/runtime/eap/jboss-eap-7.1/standalone/configuration/logging.properties
~~~
* Character 인코딩:
new String( “한글”.getBytes(“ISO8859_1”), “UTF-8” );
<%@ page contentType="text/html;charset=UTF-8" pageEncoding="UTF-8" %>
