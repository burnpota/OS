제목 : yum을 통해서 patch update 전에 사전에 의존성 문제를 확인하는 방법 문ㅇ

======================
사전문의

어떤 문제/오류/결함이 발생했습니까? 기대하시는 결과는 무엇입니까?

안녕하세요

SDS 김지민 선임입니다.

yum을 patch update 하기 전에 사전에 의존성 문제를 확인하는 방법이

있는지 문의드리려고 합니다.

예를 들면

아래와 같이 패키지가 설치되어있는데 krb5-libs-1.15.1-8.el7.x86_64 패키지만 업데이트 할경우
i686이 문제가 되었었습니다.

그래서 혹시 현재 patch하려고 createrepo를 해 놓은 묶음이 있는데 해당 묶음으로
서버에서 혹시 의존성 문제가 발생하는지 않하는지를 확인하는 방법을 문의드리려고 합니다.

root@SLDDBEDL01 <ICT Suwon Center>:/root # rpm -qa | grep -i krb5-libs
krb5-libs-1.15.1-8.el7.x86_64
krb5-libs-1.15.1-8.el7.i686

감사합니다.

안녕하세요? 레드햇 허경입니다.

아래 명령어로 개별 패키지에 대한 의존성 정보를 확인하실 수 있습니다.

yum deplist <패키지명>

그런데, 문의하신 내용과 같이 특정 라이브러리나 프로그램이 64비트 32비트가 동시에 필요한 경우에는 두가지 버전 모두 설치되어야 하고 업그레이드도 둘 다 되어야 합니다. yum install 또는 update 시 64비트 32비트 관련하여 충돌이 나는 것과 관련한 KCS 문서[1]를 고객포털에서 확인하실 수 있습니다.
이 내용도 참고하시는 것이 좋을 것 같습니다.

[1] https://access.redhat.com/solutions/158883

고맙습니다.
