rr_min_io와 rr_min_io_rq 는 같은 것으로 이해하시면 됩니다. 다만, rr_min_io는 커널 2.6.31 이전 버젼에서 사용되던 값이었으나, kernel 2.6.31버젼 이후에서는 rr_min_io_rq 값을 사용하시면 됩니다. 즉, RHEL6 부터는 rr_min_io_rq 값을 사용하시면 됩니다.

해당 속성의 기본값은 현재 1로 설정됩니다.


> Round-Robin에서 path Group내의 path를 번갈아가며 사용하기 위해 사용되는 기준값인 rr_min_io 의 단위( block / byte 등 )가 무엇인지 궁금합니다.

그리고 해당 값의 단위는 I/O Request 단위로 번갈아가며 사용되게 됩니다.


rr_min_io_rq The number of IO to route to a path before switching to the next in the same path group,
using request-based device-mapper-multipath. This setting should be used on systems running current kernels.
On systems running kernels older than 2.6.31, use rr_min_io. Default is 1


업무에 참고하시기 바랍니다.
