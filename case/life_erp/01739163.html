제목 : 시스템 Reboot 또는 Shutdown 시 Hang 상태 문의

======================
사전문의

어떤 문제/오류/결함이 발생했습니까? 기대하시는 결과는 무엇입니까?

안녕하세요.

프로젝트 시스템을 가끔 Reboot 또는 Shutdown (poweroff) 할 때 정상적으로 종료되지 않고 Hang 상태로 아무 반응없이 멈춰 버리는 현상이 발견되었습니다.

어디서 문제가 발생했습니까? 어떤 환경에서 발생했습니까?

패턴을 조사 해 보니 아래와 같이 2가지 케이스에 걸려 멈춰 있는 것을 확인하였습니다.

1. SAN SCSI 볼륨이 끊어져 I/O Error 가 나는 상태에서 reboot 할 때 unmount 시점에서 걸려 있습니다.

2. 일반적으로 정상적인 reboot or poweroff 시 아래와 같은 메세지에서 걸려 있습니다.

"Reached target Shutdown"

언제 문제가 발생했습니까? 이러한 문제가 자주 발생합니까? 반복적으로 발생합니까? 특정 시간에 발생합니까?

두번 째 케이스는 확인 해 보니 이미 비슷한 유사 사례를 https://access.redhat.com/solutions/2575881 여기에서 찾았는데요,

현재 프로젝트에서 운영 중인 시스템에서 가끔 발생하여 수동으로 cold reset 하는 경우가 종종 있습니다.

확인 부탁 드립니다.

문제 해결 기간 및 긴급도와 관련된 정보를 제공해 주시겠습니까?

sosreport 는 최근에 올렸던

sosreport-PPMSAL01SL-20161109150609.tar.xz
sosreport-LOG-PPMSAL01SL-20161109150609.tar.gz

파일로 확인 해 주시면 됩니다.

(모든 시스템의 버전이 동일 합니다.)

> 일단 이 옵션 외에 shutdown(reboot) 시 "Reached target Shutdown" 메세지에서 걸리는(Hang) 유사 사례가 있는지 확인 부탁 드립니다.

"Reached target Shutdown"이 나온 것은 해당 메시지 이후에 멈춘 것으로 보이고, 현재 이와 유사한 이슈들은 몇 가지 존재합니다.


해당 로그 이후에 시스템 셧다운은 마운트, 디바이스 그리고 프로세스들을 shutdown하도록 마무리 작업의 신호를 보내게 됩니다.
그런데 이런 과정에서 어디선가 정상저인 shutdown을 하지 못할 경우에 해당 메시지에서 멈추게 되는 사례들이 보고된 바 있습니다.

안녕하세요,
Red Hat 한진구 입니다.

우선 로그를 저희 엔지니어가 살펴본 결과, 특별히 문제의 로그를 찾기는 어려웠습니다. 그래서 추가적으로 디버깅 로그를 수집하기 위하여 아래와 같은 작업을 요청드리며 문제가 reproducing이 필요할 것 같습니다.

1. Edit /etc/default/grub to allow for more information at boot. Remove rhgb and quiet.

Change:
GRUB_CMDLINE_LINUX="crashkernel=512M rhgb quiet nmi_watchdog=0 transparent_hugepage=never elevator=deadline rdloaddriver=hpsa rdloaddriver=qla2xxx"

To:
GRUB_CMDLINE_LINUX="crashkernel=512M nmi_watchdog=0 transparent_hugepage=never elevator=deadline rdloaddriver=hpsa rdloaddriver=qla2xxx"


2. Also add systemd logging and rd.udev.debug, rd.debug, and log_buf_len=10M logging.

GRUB_CMDLINE_LINUX_DEFAULT="console=tty0 console=ttyS0,115200n8 rd.udev.debug rd.debug systemd.log_level=debug log_buf_len=10M"


해당 설정 후, 'grup2-mkconfig'로 rebuilding해주시고.. 시스템 재부팅이 필요합니다.


이후, 문제 발생시 해당 로그를 수집하여 본 케이스에 첨부하여 주시기 바랍니다.


감사합니다.
