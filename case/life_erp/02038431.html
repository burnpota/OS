Q1) 추가로 아래와 같이 multipath -ll 정보에서 아래와 같은 메세지가 발생하는데 asm/.asm_ctl_spec: No fc_host device for 'host-1' 혹시 어떤 의미인지 문의드리려고 합니다.

root@SLPAFIDL01 /root # multipath -ll
Feb 20 14:22:46 | asm/.asm_ctl_spec: No fc_host device for 'host-1'
Feb 20 14:22:46 | asm/.asm_ctl_vbg0: No fc_host device for 'host-1'
Feb 20 14:22:46 | asm/.asm_ctl_vbg1: No fc_host device for 'host-1'
~~ 생략

A1) 위의 메시지는 /dev/asm/.asm_ctl_vbg* 파일을 scan 한 뒤 multipath 가 패스정보로 이들을 더 하려고 하지만 "asm' 디바이스가 block 디바이스가 아니어서 추가할 수 없다는
경고성 메시지로 시스템에 어떤 영향이 있지는 않습니다.

아래와 같이 ASM 디바이스들을 blacklist 처리를 하셔서 메시지가 사라지는지 확인해보시고 피드백 해주시길 부탁드립니다.

$ vi /etc/multipath.conf
~~ 생략
blacklist
{ ......
devnode "^(ram|raw|loop|fd|md|dm-|sr|scd|st)[0-9]*"
devnode "^asm/*"
devnode "^ofsctl"
......
}

수정한 뒤에 이들을 반영하기 위해서 multipath 를 reload 합니다.

$ service multipathd reload


Q2) SLPAFIDL01/SLPAFIDL02 서버가 공유하는 disk이고 multipath까지 설정을 하였는데 해당 device 회수 절차를 문의드리려고 합니다.
제가 생각한 절차를 아래 기입하였는데 혹시 추가로 해야할 부분이 있는지 문의드립니다.

A2) 케이스에 올려주신 1번과정에서 multipath -f 이후에 아래의 blockdev --flushbufs 를 양쪽 노드에서 추가 실행하시고, 나머지 부분은
그대로 진행하셔도 될 거 같습니다.

$ multipath -f ext98
$ multipath -f ext99
$ blockdev --flushbufs /dev/mapper/ext98
$ blockdev --flushbufs /dev/mapper/ext99

감사합니다.
