우선, 저희 CEE 기술지원 레벨에서는 커널 파라메터의 디폴트 값이외에
튜닝과 관련된 구체적인 값을 제공드리지는 않는 점에 대해 미리 양해부탁드립니다.

업무에 도움을 드리고자 올려주신 파일을 확인해 봤습니다만,
해당 값들은 일반적으로 패킷드랍이 발생할 경우, 퍼포먼스 향상을 위하여 조정가능한 값으로 보입니다.

저희 네트워크 그룹 개발관련 전문가들이 작성한 문서에도 해당 값들을 조정하고 있으므로
CI-TEC 권고에 따라 조정하시는 자체가 특별히 문제가 될 것 같지는 않습니다.

What are the recommendations for 10 Gigabit network adapter tuning parameters?
- https://access.redhat.com/solutions/127143

다만, 최적의 튜닝값은 사용하시는 워크로드, 시스템 사양 등 실 환경에 따라 다르므로
실제 테스트를 통해 적정값인지 여부 확인 및 최적값을 도출하실 필요가 있을 것입니다.

마지막으로, net.ipv4.tcp_retries 의 경우 linux 는 이러한 파라메터를 가지고 있지 않습니다.
아마도 net.ipv4.tcp_retries2 를 말씀하시는 것 같은데요,
해당 값은 늘릴 수 있으나 늘릴 경우, TCP 연결에서 데드 커넥션이 발생할 경우 해당 소켓에 대해서 정리하는 시간이 늘어날 것입니다.
다만, 일반적으로 이번과 같이 해당 값을 늘리는 경우보다는 너무 작은 값(3)으로 조정하는 경우에 주의가 필요합니다.
~~~
tcp_retries2 (integer; default: 15; since Linux 2.2)
The maximum number of times a TCP packet is retransmitted in
established state before giving up. The default value is 15,
which corresponds to a duration of approximately between 13 to
30 minutes, depending on the retransmission timeout. The
RFC 1122 specified minimum limit of 100 seconds is typically
deemed too short.
~~~

Possible minimum tunable value for tcp_retries2 and tcp_syn_retries
- https://access.redhat.com/solutions/31043
