제목 : QLogic HBA (qla2xxx) 의 invalid_tx_word_count 증가 영향도 문의

======================
사전문의

어떤 문제/오류/결함이 발생했습니까? 기대하시는 결과는 무엇입니까?

안녕하세요.

현재 시스템에 대해 전체 전수조사를 매일 진행하고 있습니다.

물리 (Physical) 시스템 QLogic HBA (qla2xxx) Port 에서 최근 아래와 같은 상태를 감지 하였습니다.

어디서 문제가 발생했습니까? 어떤 환경에서 발생했습니까?

1. PMPRAL07SL HBA Port Check

# cat /sys/class/fc_host/host5/statistics/invalid_tx_word_count
0x41

# cat /sys/class/fc_host/host5/statistics/link_failure_count
0x1
=======================================================

2. ISLTAL03SL HBA Port Check

# cat /sys/class/fc_host/host5/statistics/invalid_tx_word_count
0xb0

# cat /sys/class/fc_host/host5/statistics/link_failure_count
0x1

언제 문제가 발생했습니까? 이러한 문제가 자주 발생합니까? 반복적으로 발생합니까? 특정 시간에 발생합니까?

이와 관련하여 SAN Switch 의 port Error show 조회도 했으나, 특이사항은 없었습니다.

또한, O/S 측면에서도 SAN Error 또는 볼륨상태 관련하여 특이사항은 없었습니다.

https://access.redhat.com/solutions/219603 솔루션을 보게 되면,

invalid_tx_word_count 값이 0x0 이 아니라면 케이블 등 하드웨어 쪽으로 체크를 해야 한다고 나와 있는데요,

이 값이 증가함에 따른 영향도를 알고 싶습니다.

그리고 invalid_tx_word_count, link_failure_count 이 의미하는 내용 확인 부탁드립니다.

감사합니다.

아래는 HBA 에 대한 정보를 확인하는 방법입니다.

$ lspci –vv | grep "Fibre Channel"

Get the HBA WWPN:

$ cat /sys/class/fc_host/host*/port_name

Get the HBA WWNN:

$ cat /sys/class/fc_host/host*/node_name

Get the HBA Speed:

$ cat /sys/class/fc_host/host*/speed

Get the HBA port state:

$ cat /sys/class/fc_host/host*/port_state

Get the HBA port TYPE:

$ cat /sys/class/fc_host/host*/port_type

Get the HBA statistics:

$ cat /sys/class/fc_host/host*/statistics

추가적으로 에러가 지속되면 HBA 혹은 FC 케이블등을 교체해보는 것도 고려해보시기 바랍니다.

현재 하루 주기적으로 시스템을 점검하고 있습니다.

어제->오늘 새벽 까지는 PMPRAL07SL 시스템의 경우 invalid_tx_word_count 값이 약 250 에서 900 가까지 증가 하였는데요,

시스템의 journal 을 확인해도 별다른 Volume 또는 Multipath, HBA 관련 에러는 발견하지 못했습니다.

그리고, 여러 시스템에서 발생하는 것이 아니고, 문의드린 2대의 시스템에서만 발견되고 있어 향후 장애 발생 가능성이 있을지 우려가 되는 상황입니다.

이와 관련하여 statistics 외에 좀 더 자세히 모니터링 할 수 있는 방안이 있는지 확인 부탁드립니다.

감사합니다.

안녕하세요,

Red Hat Global Support Services 를 이용해 주셔서 감사합니다.

invalid_tx_word_count 는 HBA 한 FC 링크상에서 decode 가 될 수 없고, 유효하지 않은 어떤 데이터를 받을 때 발생합니다.
이 값이 올라갔다는것은 HBA 와 switch 사에의 local link에 이슈가 있다는 의심해 보실수 있습니다.
해당 값은 부팅후에 reset 됩니다.
만일 하드웨어 체킹에 의하여 이슈가 없는것으로 판단되며 해당 값이 지속적으로 증가하지 않는다면 문제가 없을 겁니다.

감사합니다.
