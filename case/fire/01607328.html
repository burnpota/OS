1) 요약 : [삼성화재][보험ERP][가동계] NFS mount retry interval ( hard mode ) like timeo on client
2) 설명 :
NFS Client에서 mount를 재시도하는 간격을 조정하여, NFS 스토리지 Fail-Over시 mount에 소요되는 시간을 줄이려고 합니다.

hard 옵션으로 마운트되는 nfs 파일시스템에대하여 timeo 옵션 외에,
다른 옵션으로 mount retry interval을 조정하거나,
tcp 등의 커널 파라미터 등의 time-out값을 변경하여서도 줄일 수 있는지 알고 싶습니다.

NFS스토리지는 HITACHI HNAS 이며, 이중화 구성되어 있습니다.
/etc/fstab에 등록된 마운트 옵션은 하기와 같습니다.

vers=3,proto=tcp,hard,intr,timeo=600,retrans=2,wsize=65536,rsize=65536,_netdev

안녕하세요,

Red Hat Global Support Services 를 이용해 주셔서 감사합니다.

NFS 스토리지 Fail-Over시 mount에 소요되는 시간은 Fail-Over시 사용되는 시간에 의하여
결정됩니다.

timeo 옵션이 600(1/10초 단위)일 경우 NFS 클라이언트는 RPC timeout가 발생후 retransmission 진행하게 되며
디폴트로 0.7초 사이에 답변이 없을 경우 다음 retransmission은 2배의 시간후 즉 1.4초후에
retransmission을 진행하게 됩니다. 이렇게 RPC request가 60초동안 답변이 없을 때까지 retransmission
작업을 하게 됩니다.

따라서 NFS 스토리지 Fail-Over에 소요되는 시간을 줄이시면 마운트에 사용되는 시간이 적게 됩니다.

감사합니다.
