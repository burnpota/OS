Q1) Is it possible that we can add RRP more than one on HA environment for RHEL7?
If yes, Could you explain how to configure RRP on this environment?

A1) Yes, please review this article:
- How can I configure a redundant heartbeat network for a RHEL 5, 6, or 7 High Availability cluster?
https://access.redhat.com/solutions/61832

---

Q2) Is it possible that we can configure RRP over virtual ip network on HA environment for RHEL7?

A2) You should be able to. There are some limitations so please read above article. As long as
virtual interface appears as physical interface, bonded, etc. The one limitation related to
virtual ip is no teaming support. Also note that max is 2 rings.

- How can I configure a redundant heartbeat network for a RHEL 5, 6, or 7 High Availability cluster?
https://access.redhat.com/solutions/61832

---

Q3) Is it possible that we can configure cluster using qdisk for legacy cman type on HA environment for RHEL7?
If no, Could you explain how to migration method on this environment?

A3) There is no drop in replacement for qdisk with RHEL 7 pacemaker.

* The question to ask is why do you need qdiskd for your RHEL 7 cluster?
* What purpose or problem do you think qdisk would solve for your RHEL 7 cluster?

Answering those questions will us answer your question.


---

Q4) We know self fence(suicide) when rgmanager check status monitor for resource.
Is it possible that we can configure same operation using pacemaker on HA environment for RHEL7?
If yes, Could you explain how to configure option for pacemaker on this environment?

A4) Set the `on-fail` attribute for the resource to `fence` or another valid value for that attribute.

See the article: https://access.redhat.com/documentation/en-US/Red_Hat_Enterprise_Linux/7/html-single/High_Availability_Add-On_Reference/index.html#s1-genresourceparams-HAAR


For example when the following resource was created:
# pcs resource create clusterfs Filesystem device="/dev/cluster_vg/cluster_lv" directory="/var/mountpoint" fstype="gfs2" "options=noatime" op monitor interval=10s on-fail=fence clone interleave=true
