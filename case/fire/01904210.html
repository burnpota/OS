"BZ - 1381128 - If executing sar with Korean character set(ko_KR.UTF-8) on RHEL7, time stamp is not printed." 에 대한 패치가 되어 있어서 입니다. 패키지가 의존성이 처음 설치시 lm_sensors-libs 이 의존성이 생기는데 이후 업데이트 할때 문제가 없는지요?

3. 해당 업데이트를 온라인상에서 진행해도 문제가 없는지요? 리부팅이 꼭 필요한 업데이트 인가요?

A1. 해당 버그질라에서 확인해보았으나 7.2 EUS에는 해당 버그의 패치에 대해 아직 계획이 없는 것으로 확인하였습니다.
https://access.redhat.com/downloads/content/69/ver=/rhel---7/7.4/x86_64/product-software
에서 sysstat 패키지를 검색한후 다운받아서 업데이트 하실수 있습니다.

~~~~~~~~~~~~~~~
Q2. 만약 업데이트 해야 한다고 하면 권고 버전보다 높은 버전인 sysstat-10.1.5-12.el7.x86_64 버전을 진행할려고 합니다 .이 버전의 경우
"BZ - 1381128 - If executing sar with Korean character set(ko_KR.UTF-8) on RHEL7, time stamp is not printed." 에 대한 패치가 되어 있어서 입니다. 패키지가 의존성이 처음 설치시 lm_sensors-libs 이 의존성이 생기는데 이후 업데이트 할때 문제가 없는지요?

A2.테스트 해본결과 lm_sensors-libs가 이미 설치되어있으면 sysstat-10.1.5-12.el7.x86_64 패키지만 아래의 명령으로 올려도 문제가 없는것으로 확인했습니다.
#rpm -Uvh sysstat-10.1.5-12.el7.x86_64

~~~~~~~~~~~
Q3. 해당 업데이트를 온라인상에서 진행해도 문제가 없는지요? 리부팅이 꼭 필요한 업데이트 인가요?

A3.네 문제가 없습니다.리부팅도 필요하지 않습니다.
