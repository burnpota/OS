안녕하세요? Red Hat Technical Account Manager 송 창 안 입니다.

어제 미팅에서 안내한 seekwatcher 라는 내용이 있으며, 이 내용은 저희 성능튜닝가이드에 있는 내용이며,
IO 내용을 손쉽게 확인이 가능하게 해석해주는 툴 입니다.

6.3. 도구 (blktrace 관련 내용이 있습니다.)
https://access.redhat.com/documentation/ko-kr/red_hat_enterprise_linux/6/html/performance_tuning_guide/ch06s03

예제는 아래와 같습니다.
1. blktrace를 시스템에 설치합니다.
# yum install blktrace
2. seekwatcher를 설치 합니다.
# yum install seekwatcher
3. 해당하는 디스크에 blktrace 를 모니터링을 시작합니다.
# blktrace -d [device]
^C=== sda2 ===
CPU 0: 1 events, 1 KiB data
CPU 1: 300 events, 15 KiB data
CPU 2: 157 events, 8 KiB data
CPU 3: 0 events, 0 KiB data
Total: 458 events (dropped 0), 22 KiB data
4. seekwatcher를 이용하여 그래프 생성.
# seekwatcher -t [device]_blktrace.* // trace.png 이미지 파일이 생성 됩니다.
