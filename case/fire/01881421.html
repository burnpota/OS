제목 : NFS I/O and tcp_max_slot_table_entries parameter

======================
사전문의

안녕하세요. 정윤성책임 입니다.



효성에서 RHEL6.3이후 버전 10G 네트웍 환경에서 NFS사용시 아래내용처럼 파라미터를 수정할것을 권고 하였습니다.



해당 파라미터 수정하는 부분에 대해 다른 영향도 없는지 레드햇에서 확인 부탁드립니다.



7/8(토)부터 화재 ERP DR/검증계 안정화 작업 예정이라 7/3(월)까지 답변 부탁드리겠습니다.



# 10G 환경에서 NFS 사용시 특정 클라이언트 I/O 점유 예방을 위해 아래 내용을 권고 합니다.



RHEL6.3 에서는 버퍼에 64K 를 최대 128개 까지 채워서 전달하였으나, RHEL6.3 이후 부터는 버퍼에 64K 를 default 로 약 6만5천개 까지 채워서 보냅니다.

따라서, 10GB 환경의 특정서버에서 많이 I/O 를 수행하면 FPGA(I/O처리CPU) high load 및 다른 서버들의 I/O 처리에 영향을 미칠 수 있습니다.



Linux Client 에서 아래와 같이 조치 필요.

. On all of the nodes running RHEL 6.3 or later or Debian Jessie or later, run

sudo sysctl sunrpc.tcp_max_slot_table_entries=128

Red Hat 한진구 입니다.

해당 파라미터 값 변경은 크게 시스템에 영향도는 없는 파라미터입니다.
sunrpc.tcp_max_slot_table_entries=128

해당 파라미터는 NFS의 명령을 보내는 최대의 단위이기 때문에 NFS 서버에서 처리할 수 있는 적절한 수치가 128인 것으로 보입니다.


참고적으로 해당 파라미터에 대해서 간략히 설명을 드린다면, 한 번에 NFS Client에서 NFS Server로 보내는 명령의 최대갯수이며 RHEL6.3 이전에는 16이었던 반면에 현재는 65536까지 최대의 명령을 보낼 수 있는 값입니다.
