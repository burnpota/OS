안녕하세요,
Red Hat Technical Account Manager 한진구 입니다.

일부 타사이트에서 RHEL7.1에서 RHEL7.2로의 업데이트 후에 발견되는 IPC 이슈가 있었습니다.

간략히 설명을 드리면, IPC(semaphores, shared memory, message queues)를 사용하는 application에서 performance 등에 대한 이슈가 발생되었던 건이 있었고, IPC를 사용하는 대부분의 시스템은 일반적으로 DB그리고 SAP와 같은 application이 었습니다. 그래서 내부적인 안내를 통해서 현재 SAP 시스템들에는 해당 workaround가 적용이 사전에 된것으로 알고 있으나,

최근 Apache를 사용하는 시스템에서도 유사한 사례가 발생됨에 따라, Non-SAP 시스템들 중 IPC를 사용하는 시스템 (Apache 시스템 포함하여)에 향후 진행 중인 patch가 나오면 안내코자 사전안내 케이스를 오픈하였습니다.

현재 이슈는 bugzilla를 통해서 현재 진행중에 있으며, 현재 QA상태에 있습니다.


혹시.. 급히 업무에 적용을 해야 하는 시스템들을 위해서 아래와 같이 workaround를 제공해드립니다.


1. Edit /etc/systemd/logind.conf

# vi /etc/systemd/logind.conf
RemoveIPC=no

2. restart systemd-logind service

# systemctl restart systemd-logind.service
======================
향후, patch가 release되면 다시 안내토록 하겠습니다.

감사합니다.
