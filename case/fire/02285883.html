코어 저장시 디렉토리는 kernel.core_pattern에 의하여 결정되며 절대경로로 설정시 모든 코어 덤프 파일이
동일 디렉토리에 저장, 상대경로로 설정시 이슈가 발생되는 프로세스 경로에 저장하게 됩니다.
특정 프로세서, 특정 사용자에 대해서 별도 설정하는 방법이 없습니다.

코어가 생성되지 않는 이슈에 대하여 혹시 고객님 환경에서 abrtd 서비스가 기동되어 있는 환경 이신지요?
이외에 아래와 같은 설정을 주석처리 해주셨는지요? 확인 부탁 드립니다.
/etc/profile
# No core files by default
# ulimit -S -c 0 > /dev/null 2>&1

How to enable core file dumps when an application crashes or segmentation faults
