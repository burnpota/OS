문의 주신 내용에 대해서 답변 하도록 하겠습니다.
유선상으로 제가 잘못 전달 해드린 내용이 있으며, 정정 하도록 하겠습니다.

전제 조건은 아래와 같습니다.
* 기본적으로 서브크립션이 등록된 RHEL 시스템 혹은 RHEL이 설치된 노트북에서 local repository 작업이 가능합니다.
* internet을 통해 저희 repository로 접근이 가능해야 합니다.

ansible engien 패키지는 RHEL Extra repository에 구성이 되어 있는 것으로 판단하고 있습니다.
하지만. 제가 확인한 바로는 최소한의 아래의 의존성 패키지가 존재하는 것으로 파악이 되며, 패키지명 - repo label name 순으로 명시 하였습니다.
PyYAML - rhel-7-server-rpms
python-jinja2 - rhel-7-server-rpms
python-paramiko - rhel-7-server-extras-rpms <-----------
python-paramiko-doc - rhel-7-server-extras-rpms <-----------
python-setuptools - rhel-7-server-rpms
python-six - rhel-7-server-rpms
python2-cryptography - rhel-7-server-rpms
sshpass - rhel-7-server-extras-rpms <-----------

따라서 최소한 추가적으로 rhel-7-server-rpms, rhel-7-server-extras-rpms repository는 구성이 되어야 하며, ansible engine에 대해서 설치가 가능 합니다.
# yum install ansible

## 아래의 내용은 시스템에 local repository 구성하는 내용입니다.

1. 기본적으로 서브크립션이 등록된 시스템에서 작업이 필요 하며, 아래와 같은 패키지가 설치 되어 있어야 합니다.
# yum install yum-utils createrepo -y

2. reposync 명령어를 통해 test라는 디렉토리에 local repository에서 rhel-7-server-rpms repo-id를 구성하는 예제 입니다.
# reposync --gpgcheck -l --repoid=rhel-7-server-rpms --download_path=/test

reposync 명령어에 대한 설명은 아래와 같습니다.
# reposync --gpgcheck -l --repoid=<repo-id> --download_path=<경로명>

4. 위에서 제가 설명한 내용은 아래의 저희 KCS문서에 요약 본이며, 자세한 설명은 아래의 링크를 참조 하시길 바랍니다.

How to create a local mirror of the latest update for Red Hat Enterprise Linux 5, 6, 7 without using Satellite server?
https://access.redhat.com/solutions/23016

5. local repository가 만들어 지면, 그 내용을 설치하려는 시스템에 가져와서 아래와 같이 구성 후 yum 을 이용한 설치에 대한 테스트를 진행 합니다.
아래는 예제에 해당이 됩니다.
# cat /etc/yum.repos.d/rhel7-internal.repo

[internal-repo]
name=Internal Repository
baseurl=file:///repos/test
enabled=1
gpgcheck=0

6. 아래 내용은 local cdrom에서 local repository 사용하는 예제에서 baseurl부분을 참조 하였습니다.

Need to set up yum repository for locally-mounted DVD on Red Hat Enterprise Linux 7
https://access.redhat.com/solutions/1355683
