안녕하세요, Red Hat Technical Account Manager 송 창 안 입니다. 지난 11월 1일 공개된 보안 취약점 관련 안내입니다.

Linux 커널의 특수하게 조작된 TCP 패킷을 처리하는 방식에서 SegmentSmack이라는 취약점이 발견되었습니다.
이 취약점을 사용하여 공격자는 원격에서 시간과 계산식이 많이 소요되는 호출을 tcp_collapse_ofo_queue() 및 tcp_prune_ofo_queue() 함수로 전송시킬 수 있게 됩니다.
이 과정에서 특수하게 조작된 패킷을 현재 진행 중인 TCP 세션으로 보내게 되면 들어오는 네트워크 트래픽의 상대적으로 작은 대역폭으로 인해 CPU에서 과부하가 발생하고 Denial of Service attack 즉
DOS공격이 가능하게 됩니다. 최악의 경우 공격자는 공격 트래픽이 2 kpps 미만의 영향을 받는 호스트 또는 장치를 지연시킬 수 있습니다.

* CVE버젼 :
- https://access.redhat.com/security/cve/cve-2018-5390
- https://access.redhat.com/security/cve/cve-2018-5391

* 대상 버전:
- Red Hat Enterprise Linux Server 7 및 파생 버젼
- Red Hat Enterprise Linux Server 6 및 파생 버젼
- Red Hat Virtualization 4
- Red Hat Enterprise MRG 2

* 관련 패키지: kernel

* 영 향 도
1) 심각도 등급 : 중요 (Important)

2) 영향 조건
- 영향에 관련 패키지 업데이트

3) 완화 방법
- 패키지 업데이트 (현재 진행중 입니다)
- https://access.redhat.com/ko/articles/3684551 (국문을 통해 완화 방법 소개)
취약점에 완화가 적용된 커널을 설치하는 것 외에 net.ipv4.ipfrag_high_thresh 및
net.ipv4.ipfrag_low_thresh (및 해당 IPv6 net.ipv6.ipfrag_high_thresh 및 net.ipv6.ipfrag_low_thresh 포함)
sysctl 매개 변수의 디폴트 값을 변경하여 해당 취약점을 완화
- 그외 질문 사항들에 대해서는 케이스를 통해 질문을 주시는 경우, 확인 후 답변하도록 하겠습니다.

해당 보안 취약점은 서비스에 영향이 없도록 업데이트 일정을 계획하시어 해당 패키지에 대해서 업데이트를 권장 드립니다.
