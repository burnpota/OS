메시지 1:
Aug 12 13:41:16 eccalp03 kernel: i8042: Can't read CTR while initializing i8042
Aug 12 13:41:16 eccalp03 kernel: i8042: probe of i8042 failed with error -5

상기 메시지는 i8042 드라이버와 BIOS 사이의 버그인것으로 보고되어 있으며 이는 BIOS에서 "USB legacy support"
옵션을 비활성화 하거나 grub에 커널 파라미터 usb-handoff를 추가하셔서 해소하실수 있습니다.

메시지 2:
Aug 12 13:59:09 eccalp03 kernel: megaraid_sas: module verification failed: signature and/or required key missing - tainting kernel

상기 메시지는 레드햇이 아닌 다른 3rd party 모듈이 서버에 로딩되었을 경우 기록되는 메시지로 확인됩니다.

참고문서:

~~~
/**
* mei_quirk_probe - probe for devices that doesn't valid ME interface
*
* @pdev: PCI device structure
* @ent: entry into pci_device_table
*
* returns true if ME Interface is valid, false otherwise
*/
static bool mei_me_quirk_probe(struct pci_dev *pdev,
const struct pci_device_id *ent)
{
u32 reg;
/* Cougar Point || Patsburg */
if (ent->device == MEI_DEV_ID_CPT_1 ||
ent->device == MEI_DEV_ID_PBG_1) {
pci_read_config_dword(pdev, PCI_CFG_HFS_2, &reg);
/* make sure that bit 9 (NM) is up and bit 10 (DM) is down */
if ((reg & 0x600) == 0x200)
goto no_mei;
}

/* Lynx Point */
if (ent->device == MEI_DEV_ID_LPT_H ||
ent->device == MEI_DEV_ID_LPT_W ||
ent->device == MEI_DEV_ID_LPT_HR) {
/* Read ME FW Status check for SPS Firmware */
pci_read_config_dword(pdev, PCI_CFG_HFS_1, &reg);
/* if bits [19:16] = 15, running SPS Firmware */
if ((reg & 0xf0000) == 0xf0000)
goto no_mei;
}

return true;

no_mei:

return false;
}
