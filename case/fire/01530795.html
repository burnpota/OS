안녕하세요,
Red Hat Technical Account Manager 한진구 입니다.


금일 언론에 보도되고 최근에 NTP에 리포트된 다수의 보안 취약점과 관련하여 사전 안내드립니다.


* NTP 보안 취약점 내용 (13개 종류, 16개 CVE)

1) CVE-2015-7871 : cryto-NAK을 통해 인증을 우회하는 취약점
2) CVE-2015-7855 : decodenetnum()함수에서 위조된 값에 대해 FAIL 반환하는 대신 오류 값을 발생하는 취약점
3) CVE-2015-7854 : 패스워드 길이 처리하는 과정에서 발생하는 메모리 손상 취약점
4) CVE-2015-7853 : refclock 드라이버에서 부적절한 데이터 길이로 인해 발생하는 버퍼오버플로우 취약점
5) CVE-2015-7852 : ntpq atoascii()함수에서 발생하는 메모리 손상 취약점
6) CVE-2015-7851 : saveconfig에서 발생하는 디렉토리 접근 취약점
7) CVE-2015-7850 : NTP의 원격 구성 기능에서 발생하는 DoS 취약점
8) CVE-2015-7849 : tursed key에서 발생하는 use-after-free 취약점
9) CVE-2015-7848 : 조작된 패킷에 의해 7 loop counter를 처리할 때, 발생할 수 있는 Out-Of-Bounds 취약점
10) CVE-2015-7701 : CRYPTO-ASSOC에서 발생하는 메모리 손상 취약점
11) CVE-2015-7703 : "pidfile"과 "driftfile"만 허용 가능한 디렉토리 접근 취약점
12) CVE-2015-7704, CVE-2015-7705 : timestamp 유효성 검사를 하는 클라이언트에서 발생할 수 있는 DoS 취약점
13) CVE-2015-7691, CVE-2015-7692, CVE-2015-7702 : autokey 데이터 패킷 길이 체크 시 발생하는 취약점

* 향후 대응 방안 안내
- 이미 패치된 CVE-2015-7704의 경우 중요도가 Important여서 보다 신속하게 패치되고 릴리즈 되었습니다.
- 현재 10개의 추가 패치가 진행중이며 이들은 Moderate, Low 중요도여서 다소 시간이 소요될 수 있습니다.
- 따라서 외부에 노출된 NTP 서버가 있으시다면 CVE-2015-7704에 의거 패치를 신속히 진행하시고,
- 그렇지 않을 경우 관리상의 효율성을 위해 10개의 패치가 완료된 후 일시에 적용하시는 것도 방법일 수 있습니다.
- 계속해서 이슈 모니터링 하겠습니다.
