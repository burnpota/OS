안녕하세요,

Red Hat Global Support Service 를 이용해주셔서 감사합니다.

문의하신 내용에 대하여 각 각 답변드립니다.

Q1) lvm 으로 스트라핑 구성시 i/o 성능 향상을 할수 있는지요?
A1) 네 가능합니다. 아래는 stripe lv 구성시 성능면에서 얻을 수 있는 장점입니다.

- Large sequential reads / writes 작업시에 I/O 효율성이 증대 될 수 있습니다.
- data writing 시 가지고 있는 개수만큼의 물리볼륨에 Round-Robin 방식으로 쓰기 때문에 성능이 향상 될 수 있습니다.
- striping 을 이용하면 I/O 가 병렬처리 될 수 있습니다.
- 어떤 상황에서는 물리 볼륨의 개수가 추가됨에 따라 꾸준한(linear) 성능향상을 얻을 수 있습니다.

이와 같은 내용은 첨부 문서 [1] 에서 찾으실 수 있으며, 아래는 원문입니다.

When you write data to an LVM logical volume, the file system lays the data out across the underlying physical volumes. You can control the way the data is written to the physical volumes by creating a striped logical volume. For large sequential reads and writes, this can improve the efficiency of the data I/O.
Striping enhances performance by writing data to a predetermined number of physical volumes in round-robin fashion. With striping, I/O can be done in parallel. In some situations, this can result in near-linear performance gain for each additional physical volume in the stripe.

Q2) 구성시 주의 해야할 조건이 있는지요?
A2) 구성시 특별한 요건에 대해서는 확인되지 않습니다.

Q3) 추후 추가 확장시 제한 조건이 있는지요?
A3) stripe lvm 도 lvm 이기 때문에 lvm 이 가지고 있는 제한 조건에 영향을 받습니다. lvm 제한조건에 대해서는 첨부 문서 [2] 를 참조 부탁드립니다.
추가로 확장시에는 최초 stripe lv로 구성된 물리볼륨의 개수만큼씩 확장하여야 합니다.
예를 들면 최초에 2 개의 물리 볼륨으로 stripe lv 로 구성하였다면, 이 이후에는 2개씩 물리볼륨을 추가하여 확장해야합니다. 이에 대한 내용은 첨부 문서 [3] 에서 보실 수 있습니다.

Q4) 안정성에 제한이 있는지요?
A4) RHEL 7.0 ~ RHEL 7.5 의 Release Note 의 known issue 를 찾아보았으나 stripe lvm 관련 버그는 찾을 수 없었습니다.
안정성 관련 구체적 사례를 말씀해주시면 관련된 case 가 있는지 확인해보도록 하겠습니다.

[1] "2.3.2. Striped Logical Volumes"
https://access.redhat.com/documentation/en-us/red_hat_enterprise_linux/7/html-single/logical_volume_manager_administration/#striped_volumes

[2] What is maximum size of logical volume?
https://access.redhat.com/solutions/65201

[3] 4.4.17. Extending a Striped Volume
https://access.redhat.com/documentation/en-us/red_hat_enterprise_linux/7/html-single/logical_volume_manager_administration/#stripe_extend
