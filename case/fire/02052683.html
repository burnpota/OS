OUTPUT=ISO
OUTPUT_URL=file:///osbackup
BACKUP=NETFS

BACKUP_PROG_EXCLUDE=("${BACKUP_PROG_EXCLUDE[@]}" '/data1' ' '/data2' '/data3' '/media' '/var/tmp' '/var/crash' '/CRASH' '/dev/' '/sys' '/proc' '/mnt' '/tmp' '/run' '/media' '/lost+found' '/osbackup' )
NETFS_KEEP_OLD_BACKUP_COPY=y
ONLY_INCLUDE_VG=( "rhel" )

따라서 추가적으로 EXCLUDE_VG= 옵션으로 vg01, vg02 를 지정할 필요가 없습니다.

'recover -d -v mkbackup' 수행전에 /etc/fstab에 정확하게 마운트포인트가
기재되었는지 확인이 필요합니다.

예:

# grep data /etc/fstab

/dev/vg01/testlv1 /data1 ext4 defaults 0 0
/dev/vg02/testlv2 /data2 ext4 defaults 0 0

# rear -d -v mkbackup

rear 설정파일에서는 아래와 같이 사용하고 있습니다.

OUTPUT=ISO
OUTPUT_URL=file:///osbackup
BACKUP=NETFS
BACKUP_URL=file:///osbackup
BACKUP_PROG_EXCLUDE=("${BACKUP_PROG_EXCLUDE[@]}" '/data1' ' '/data2' '/data3' '/media' '/var/tmp' '/var/crash' '/CRASH' '/dev/' '/sys' '/proc' '/mnt' '/tmp' '/run' '/media' '/lost+found' '/osbackup' )
NETFS_KEEP_OLD_BACKUP_COPY=y
ONLY_INCLUDE_VG=( "rhel" )


rear 복원 수행시 /data1~/data3 의 파티션이 모두 사라져 버려서 해당 파티션을 다시 만들지 않고 기존상태로 유지 하고 싶은데 어떻게 해야 하나요?

현재
EXCLUDE_RECREATE=( "${EXCLUDE_RECREATE[@]}" "fs:/data1" "fs:/data2" "fs:/data3")

옵션을 주고 테스트를 할려고 합니다.
