- 관련 패키지 : kernel, dracut(RHEL7), microcode_clt, linux_firmware (rhel7), qemu-kvm, qemu-kvm-rhev, libvirt
- 취약점 내용 : 최신 마이크로프로세서와 x86(인텔 및 AMD 칩셋), POWER 8, POWER 9, System z, ARM을 포함하는 주요 하드웨어 플랫폼의 운영 시스템에 영향을 끼치며 권한이 없는 사용자 계정에서 메모리에 접근해 정보를 읽을 수 있는 보안 이슈입니다.
이는 세 가지 유일한 공격 경로를 갖고 있으며 이는 공격자가 보안 장벽을 뚫고 메모리에 접근할 수 있는 사이드-채널 공격을 수행할 수 있습니다. 이번 이슈에 대한 3가지 CVE (Common Vulnerability and Exposures) 번호가 포함되었으며,
아래는 요약된 내용입니다.

* CVE-2017-5753(Spectre)는 분기 중 경계 검사(bounds checking) 취약점입니다. [bounds check bypass - variant 1]
* CVE-2017-5715(Spectre)는 간접적 탐색 악성 공격으로 데이터 유출을 일으키는 취약점이며 이를 통해, 가상화 게스트가 호스트 시스템의 메모리를 읽을 수 있도록 허용되는 취약점입니다. - [branch target injection - variant 2]
* CVE-2017-5754(Meltdown)은 투기적 캐시(speculative cache) 로딩을 사용하여 로컬에 인가되지 않는 공격자가 메모리의 내용을 읽을 수 있는 취약점입니다. - [rogue data cache load - variant 3]

자세한 내용을 확인하기 위해서는 아래 링크에서 개요 와 영향 탭을 참조해주시길 바랍니다.

https://access.redhat.com/security/vulnerabilities/speculativeexecution
