안녕하세요, Red Hat Technical Account Manager 송 창 안 입니다. 지난 2월 11일 공개된 관련 안내입니다.

악의적인 컨테이너가 호스트 시스템에서 root 레벨 액세스 권한을 얻을 수 있도록 허용하는 결함이 runc에서 발견되었습니다.
runc에서 발견된 취약점으로 인해 악의적인 컨테이너를 허용하여 호스트 시스템에서 root 레벨의 액세스 권한을 얻을 수 있습니다.
이 취약점은 Red Hat Enterprise Linux 7에서 제공되는 docker 및 runc 패키지에 영향을 미치며 Extras 채널을 통해 제공됩니다.
OpenShift Container Platform (OCP) 3.x는 Red Hat Enterprise Linux 7 Extras의 패키지에 따라 다르게 영향을 받습니다.

* CVE버젼:
- https://access.redhat.com/security/cve/cve-2019-5736

* 대상 버전:
- Red Hat Enterprise Linux 7.x 파생버젼
- Red Hat OpenShift Container Platform 3.x 파생버젼
- Red Hat OpenShift Online*
- Red Hat OpenShift Dedicated*

*OpenShift Online 및 OpenShift Dedicated는 강제 모드로 SELinux를 사용하기 때문에 악용될 위험이 없습니다.

* 관련 패키지: runc, docker

* 영 향 도
1) 심각도 등급 : 중요 (Important)

2) 영향 조건
- 영향에 관련 패키지
- https://access.redhat.com/ko/security/vulnerabilities/3902351(국문) 에서 진단 탭에서 확인하여,
https://access.redhat.com/sites/default/files/cve-2019-5736--2019-02-11-1257.sh
스크립트 다운로드 후, 영향 관련 확인.

3) 완화 방법
- ELinux를 강제 모드 (enforcing mode)로 사용함으로써 이러한 취약점이 악용되는 것을 완전히 차단
- 해당 패키지 업데이트
- https://access.redhat.com/ko/security/vulnerabilities/3902351(국문)을 통해, 문제 해결 방법 확인
- 그외 질문 사항들에 대해서는 케이스를 통해 질문을 주시는 경우, 확인 후 답변하도록 하겠습니다.
