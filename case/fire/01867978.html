[사전 안내] sudo : 부적절한 get_process_ttyname()함수를 이용한 로컬 권한 상승 (CVE-2017-2636)
========================================================================================

안녕하세요,
Red Hat Technical Account Manager 허경 입니다. 5월 30일 공개된 CVE-2017-1000367 관련 안내입니다.

* CVE 버전 : https://access.redhat.com/security/cve/cve-2017-1000367
- 대상 버전 : Red Hat Enterprise Linux 5,6,7 및 파생 버젼
- 관련 패키지 : sudo
- 취약점 내용 :

다른 사용자의 보안 권한과 관련된 프로그램을 구동할 수 있게 해주는 유틸리티인 sudo에서 보안 취약점이 발견되었습니다.
sudo 유틸리티는 get_process_ttyname()함수를 통해 시스템에 있는 /proc 파일 시스템의 프로세서 상태 파일에서 sudo 프로세스의 제어 터미널에 대한 정보를 얻도록 구현되어 있습니다.
그 결과, SELinux가 활성화된 시스템에서 sudo 유틸리티를 실행할 수 있는 권한이 있는 로컬 사용자가 위에서 언급한 보안 취약점을 이용하여, root 권한으로 상승 가능합니다.

* 영 향 도

1) 심각도 등급 : 중요 (Important)

2) 영향 조건
- https://access.redhat.com/sites/default/files/cve-2017-1000367.sh 에서 제공하는 cve-2017-1000367.sh 를 이용하여 시스템 영향 여부를 확인할 수 있습니다.
- sudo 패키지 업데이트
- https://access.redhat.com/security/vulnerabilities/3059071 에서 추가 정보를 제공하고 있습니다.

3) 각 버젼 별 패키지 목록

3-1) RHEL 5
Errata : http://rhn.redhat.com/errata/RHSA-2017-1381.html
- sudo-1.7.2p1-30.el5_11

3-2) RHEL 6
Errata : http://rhn.redhat.com/errata/RHSA-2017-1382.html
- sudo-1.8.6p3-28.el6_9.x86_64

3-3) RHEL 7
Errata : http://rhn.redhat.com/errata/RHSA-2017-1382.html
- sudo-1.8.6p7-22.el7_3.x86_64

해당 패치는 시스템 Downtime이 필요한 작업이며, 현재 해당 보안 취약점은 로컬에서 루트의 권한을 획득할 수 있는 보안취약점입니다.
서비스에 영향이 없도록 업데이트 일정을 세워서 해당 패치 버전으로 업데이트하실 것을 권장합니다.
