안녕하세요,
Red Hat Technical Account Manager 한진구 입니다.

현재 보내주신 sosreport와 구축을 담당하고 엔지니어의 정보를 토대로 Architecture Review를 한 결과를 아래 답변 드립니다.
아래 답변은 어제 삼성생명/삼성화재에서 미팅을 통해서 전달한 내용과 기본적인 골격은 같습니다.

우선 Red Hat Cluster를 정상적으로 사용으로 원활한 서비스 및 기술지원을 위해서는 기본적으로 Quorum과 Fencing을 사용을 하여야만 합니다.

하지만 현재 고객사의 구성을 검토한 결과, 해당 구성에서는 Shared Storage가 존재하지 않으며 또한 SCSI Fencing의 기본 목적인 cluster운영중 data를 보호하기 위한 목적에 부합되는 조건에 해당됩니다.
현재와 같은 구성 설정은 Red Hat 기술지원이 불가능한 클러스터 구성입니다.

현재의 구성에서 원활한 서비스를 위해서는 SCSI Fencing대신 Power Fencing을 구성하여 사용하시는 것을 권장드리며, 기본 2노드 클러스터 환경에서는 Heartbeat 네트워와 Fencing 네트워크를 같은 네트워크에 구성하여 사용하시는 것을 권고 드립니다.

HA 구성은 진행중이며,
구성이 최종 완료되기전에 문제가 될 수 있는 부분을 검토하고 기술지원이 가능한 구성을 확인 하고자 합니다.

1) HA구성 중 완료된 항목
-pacemaker cluster auth
-pacemaker cluster setup
-pacemaker cluster resource create 일부 (IPaddr)

2) HA구성 중 진행중인 항목
-pacemaker cluster fence_scsi 생성
-pacemake cluster resource create (SAP)

HA 아키텍쳐 리뷰에 필요한 사항은 추가 기술할 예정입니다.
