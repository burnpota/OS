======================
사전문의

어떤 문제/오류/결함이 발생했습니까? 기대하시는 결과는 무엇입니까?

1) 요약 : Do I need a reboot after updating openssl and java packages ?
2) 설명 :
보안권고( CVE-2016-0800, CVE-2016-0636 )에따라, OpenSSL과 JAVA 업데이트를 수행하고자 합니다.
업데이트 적용시 rebooting 필요여부에대한 공식 답변과,
rebooting 필요하지 않다면 상관없지만, rebooting이 필요하다면 왜 필요한지 이유에대한 설명을 요청 합니다.

검토해본 레드햇 문서는 하기 URL의 문서입니다.
https://access.redhat.com/solutions/27943

시스템에서 openssl 라이브러리를 사용하고 있는 첨부파일 20160334-OpenSSL_UPDATE.txt에 포함되어 있습니다.

안녕하세요

Red Hat Global Support Services 를 이용해 주셔서 감사합니다.

패키지 업데이트 이후에 반드시 리부팅이 필요하지는 않습니다만, 새로운 패키지의 적용을 위해서
리부팅을 권고드립니다.

만일, 리부팅이 가능하지 않은 시스템의 경우, openssl 라이브러리를 사용하는 서비스를 재기동하여
새로이 설치된 패키지를 이용하여 서비스가 시작될 수 있도록 조치하시기 바랍니다.

java 패키지의 경우 역시도 패키지 업데이트 이후에 java 프로세스를 재기동 하시기 바랍니다.
