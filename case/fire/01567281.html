Red Hat Technical Account Manager 한진구 입니다.


Q1) Round-Robin에서 path Group내의 path를 번갈아가며 사용하기 위해 사용되는 기준값인 rr_min_io 및 rr_min_io_rq 의 단위( block / byte 등 )가 무엇인지 문의 드립니다.

A1. 우선 rr_min_io와 rr_min_io_rq 는 같은 것으로, rr_min_io는 커널 2.6.31 이전 버젼에서 사용되던 값이었으나, kernel 2.6.31버젼 이후에서는 rr_min_io_rq 값을 사용하시면 됩니다. 즉, RHEL6 부터는 rr_min_io_rq 값을 사용하시면 됩니다.

해당 속성의 기본값은 현재 1로 설정됩니다.

그리고 문의주신 해당 값의 단위는 I/O Request 단위가 됩니다.

rr_min_io_rq The number of IO to route to a path before switching to the next in the same path group,
using request-based device-mapper-multipath. This setting should be used on systems running current kernels.
On systems running kernels older than 2.6.31, use rr_min_io. Default is 1
======================

2) path Group의 일부 path 단절시, 즉시 유효한 path로 I/O를 fail-over 하기위한 옵션이 있는지 문의 드립니다.

multipath.conf 설정 속성에 no_path_retry 라는 속성값을 'fail'로 사용하시면 path 단절시 즉시 유효한 path로 I/O를 Fail-over할 수 있습니다.


예를들아,

defaults {
...
features 0
no_path_retry fail
...
}
