$ sysctl -qa | grep "kernel.hung_task_timeout_secs"

- kernel.hung_task_timeout_secs에 대하여

Detect Hung Task Kernel thread (khungtaskd)라는 kernel thread 는 process가
D-State(Uninterruptible Sleep)에 빠져있는 시간이 설정 시간을 초과하게 되면 해당 thread에
탐측되여 로그에 찍히게 됩니다.

khungtaskd 는 시스템에 있는 모든 프로세스를 모니터링하게 됩니다. 프로세스가 D-state에 빠져있는
시간이 kernel.hung_task_timeout_secs에 설정된 값(default 120초)보다 크게 되면 메시지에
해당로그들을 뿌리게 됩니다.

일반적으로 해당로그는 시스템이 disk나 memory 소모량이 많아서 resource 부족으로 인해
process들이 정상적으로 운영되지 못하는것을 알립니다. 이로 인해 로그들이 나타난다면 운영상 심각한
문제가 발생하거나 시스템도 운영에 영향을 주지 않지만 resource 부족 문제가 해결되기전까지 시스템의
performance에 영향이 있을 것 입니다.
그리고 해당 로그들은 process의 deadlock 또는 커널 버그로 인해 나타날수도 있습니다.

만일 이와 같은 현상이 일시적으로 발생되는것이라면 무시하셔도 될것으로 보입니다.

만일 지속적으로 이슈가 발생 된다면 원인 분석을 위하여 VMcore가 필요합니다.

시스템에 kdump를 설정하시고 반드시 테스트를 진행하신후 "...in hung state for 120 seconds..." 메시지가 나타날 때,

# echo 1 > /proc/sys/kernel/hung_task_panic

를 실행하시면 VMcore가 생성됩니다. 생성된 vmcore를 dropbox로 올려주시면 분석하고 안내해드리게 됩니다.
