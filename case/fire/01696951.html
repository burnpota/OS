안녕하세요,

Red Hat Global Support Services를 이용 해 주셔서 감사합니다.


Xeon E7 v3/Xeon E5 v3/Core i7 Power Control Unit 장비에서 부팅시
" [Firmware Bug]: reg 0x10: invalid BAR (can't size)"메시지가
발생 하는 건에 대해 레드햇에서 이미 오픈 된 버그가 있습니다.

Bug 1371938 - [DELL 7.3 BUG] var/log/messages shows:kernel: invalid BAR (can't size)

해당 버그는 현재 NOTABUG 상태로 클로즈 되었으며 마지막 코멘트 내용을 보시면
BIOS에서 해당 디바이스를 hide 시켜야 된다는 의견이 있습니다.


~~~
> Is there any plan to incorporate patch ?

No, the patch is not yet upstream - RHEL does not accept patches until they are upstream - and the kernel development window for the next release closed quite a while ago.


As expressed in Intel's errata http://www.intel.com/content/dam/www/public/us/en/documents/specification-updates/xeon-e5-v3-spec-update.pdf - look for the "HSE43" entry - the expected behaviour is for the platform's BIOS to "hide" this device. That seems like the most base, correct, approach [ since we both know that HW never fixes their problems ;) ] to solving this properly. Do you disagree?

When you contact the BIOS team about this I'd be interested in knowing if they missed this again (see RHBZ 1260075) or if Intel's Premier BIOS reference code that vendor's start with somehow dropped it from their previous version(s)?
~~~

따라서 Dell쪽으로도 문의 요청을 하여 관련 해서 픽스 방안 존재여부를 확인 부탁드립니다.

안녕하세요,

Red Hat Global Support Services를 이용 해 주셔서 감사합니다.


말씀 하신 "[Firmware Bug]: reg 0x10: invalid BAR (can't size)" 에러는 아래와 같은
장비에서 발생 하고 있는 것으로 보입니다.


[ 1.660547] pci 0000:ff:1e.3: [8086:2fc0] type 00 class 0x088000
[ 1.660556] pci 0000:ff:1e.3: [Firmware Bug]: reg 0x10: invalid BAR (can't size)

[ 1.667626] pci 0000:bf:1e.3: [8086:2fc0] type 00 class 0x088000
[ 1.667635] pci 0000:bf:1e.3: [Firmware Bug]: reg 0x10: invalid BAR (can't size)

[ 1.674818] pci 0000:7f:1e.3: [8086:2fc0] type 00 class 0x088000
[ 1.674827] pci 0000:7f:1e.3: [Firmware Bug]: reg 0x10: invalid BAR (can't size)

[ 1.681493] pci 0000:3f:1e.3: [8086:2fc0] type 00 class 0x088000
[ 1.681501] pci 0000:3f:1e.3: [Firmware Bug]: reg 0x10: invalid BAR (can't size)


pci 번호 8086:2fc0 통해 해당 장비는 Xeon E7 v3/Xeon E5 v3/Core i7 Power Control Unit 로
확인 되었습니다.

Device 8086:2fc0
Name: Xeon E7 v3/Xeon E5 v3/Core i7 Power Control Unit


Intel 문서를 확인 해 보니 해당 장비는 커널에서 Power Control Unit 장비의 CONFIG_TDP_NOMINAL
정보를 base address register 로 잘못 인식 하여 booting 시 이와 같은 메시지가 발생 하였습니다.

http://www.intel.com/content/dam/www/public/us/en/documents/specification-updates/xeon-e5-v3-spec-update.pdf (HSE43 참조)


해당 버그에 대해 upstream에서 인식 하고 있으며 버그를 픽스 하였으며 kernel 3.19에서 패치 포함 되어
있다고 합니다.

https://bugzilla.kernel.org/show_bug.cgi?id=85991


레드햇에서 아직 backport 정보가 없어 보여 버그를 보고 할 예정이며 만약 관련 하여 업데이트 사항이
있으면 공유 드리도록 하겠습니다.


이와 같은 메시지는 일반적으로 시스템에 영향을 주지 않는 것으로 보이며 만약 사용중인 시스템이 해당
메시지로 인하여 영향을 받고 있다면 알려 주시기 바랍니다.
======================

감사합니다.
