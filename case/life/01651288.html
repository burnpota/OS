안녕하세요,

Red Hat Global Support Service를 이용해주셔서 감사합니다.

올려주신 내용은 RHEL7 에서 제공되는 Consistent Network Device Naming 기능을 비활성화하는
방법을 남겨주신 것으로 보입니다.

그와 관련된 KBase 는 다음과 같습니다.

How to use original ethX style NIC network interface device names in RHEL7 (disable Consistent Network Device Naming)
- https://access.redhat.com/solutions/283233

상기 KBase 를 보시면 아시겠지만, 해당 방법은 저희가 권장드리는 방법이 아닙니다.
해당 방법으로 디바이스명을 ethX 스타일로 리네임할 경우, 재부팅시 특정 NIC 와 eth 명을 고정시키기 어렵거나
하기 안내해드릴 KBase의 문제가 발생할수 있습니다.
이와 관련된 버그질라들이 오픈되었으나 이는 RHEL7의 설계 변경으로 인한 것으로 수정하지 않을 것으로 결정되었습니다.

Why RHEL7 system is failing in renaming the interfaces with error "systemd-udevd: error changing net interface name ethX to ethY: File exists" ?
- https://access.redhat.com/solutions/1422203

따라서, 고객께서 NIC 장치들을 원하시는 순서와 이름으로 리네임하시길 원하신다면
대안으로 ethX 형식의 이름이 아닌 다른 이름을 선택하여 사용하시길 바랍니다.

예)
peth0, peth1, peth2..
neth0, neth1, neth2...
net0, net1, net2..

감사합니다.
