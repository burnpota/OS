안녕하세요,

1. filematch 등을 이용한 cache에 대하여 명시적인 선언이 없을 경우 client browser에 caching을 하는지의 여부
2. 만약 cache 한다면 어떤 조건에 의해 파일 변경을 체크하여 cache이 변경되는지 확인요청드립니다.
===>
browser cache는 httpd에서 사용하는 것이 아니고, 각 browser에서 사용을 하는 것입니다.
기본적으로(디폴트로) browser에서 caching 기능은 enable 되어 있으며, web server에서 Cache-Control을 통해 HTTP header 정보를 수정하여 이러한 cache 여부를 customizing 할 수 있는 것입니다.

관련하여 참고하기 좋은 link [1] 찾아 드립니다.
각 browser에서의 caching 알고리즘은 link [2]에 나와 있는 내용을 참고하실 수 있습니다.


3. client browser에 모든 파일에 대한 caching을 원치 않을 경우 설정가이드
===>
KCS [3], [4]를 참조하실 수 있습니다.

ex) httpd가 /var/www/xxx를 DocumentRoot로 할 경우, httpd에서 서비스하는(DocumentRoot에 있는) 파일들에 대해 no-cache 설정
~~~~~
<Directory /var/www/xxx>
FileEtag None
RequestHeader unset If-Modified-Since
Header set Cache-Control "no-cache, no-store, max-age=0"
</Directory>
~~~~~

감사합니다.
