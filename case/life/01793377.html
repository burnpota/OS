0, 번들 http server 기동 시 일반적으로 systemctl restart httpd.service 로 하는 것이 메뉴얼상 권장하는데
root 계정이 아닌 일반계정으로는 실행이 불가능한 것으로 알고 있습니다.

네, 맞습니다.

1, /usr/sbin/apachectl start를 이용한 기동과 차이점이 있는지 확인 요청드립니다.

/usr/sbin/apachectl 파일을 확인해 보면 해당 명령어를 실행시에도 동일하게 systemctl 명령어를
실행합니다. 그리하여 해당 명령어도 root 계정으로 실행해 주어야 합니다.

start|stop|restart|status)
/usr/bin/systemctl $ACMD httpd.service
ERROR=$?
;;
graceful)
/usr/bin/systemctl reload httpd.service
ERROR=$?
;;
graceful-stop)
/usr/bin/systemctl stop httpd.service
ERROR=$?
;;
