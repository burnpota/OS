Q1) disktodo.conf 부분에 todo와 done 부분이 있는데 혹시 todo가 백업이 되는 부분이고 done은 백업이 되지 않는(제외) 부분인지 문의드립니다.
A1) 네 그렇습니다. 이와 관련된 내용은 아래 파일에서 확인하였습니다.

# cat /usr/share/rear/layout/prepare/default/310_remove_exclusions.sh | grep DONE
...
# Excluded components are marked as DONE in the disktodo file.
...


Q2) disklayout.conf에서는 #이 없는 부분만 layout에 반영이 되는 것인지 문의드리려고 합니다.
A2) 네 그렇습니다. 이 내용은 이전에 드린 URL 에서 확인하실 수 있습니다.

"Relax-and-Recover has reasonable defaults when creating the recovery information.
It has commented out the two external disks and any component that’s part of it.
The reason is that no mounted filesystem uses these two disks. After all, you don’t want to recreate your backup disk when you’re recovering your system."

https://github.com/rear/rear/blob/master/doc/user-guide/06-layout-configuration.adoc

Q3) diskdeps.conf 부분의 역할에 대해서도 문의드립니다.
A3) 해당 파일은 /var/lib/rear/layout/disklayout.conf 을 통해서 찾아낸 디스크들 사이에서 의존성 리스트를 저장하는 파일입니다.

# cat /usr/share/rear/lib/layout-functions.sh
...
# Generate dependencies between disks as found in $LAYOUT_FILE.
# This will be written to $LAYOUT_DEPS.
...


Q4) 구성 중에 아래와 같이 efibootmgr programs가 없다고 error가 발생하였는데 해당 부분의 발생 원인에 대해서도 문의드리려고 합니다.
A4) 해당 이슈와 일치하는 이슈가 확인되었습니다. 설치된 rear 패키지 버전이 rear-2.00-3.el7_4.x86_64.rpm 보다 낮다면 아래의 Errata 를 방문하셔서 업데이트 후 증상 재현 여부를 확인부탁드립니다.

"RHBA-2017:2937 - Bug Fix Advisory"
https://access.redhat.com/errata/RHBA-2017:2937

감사합니다.
