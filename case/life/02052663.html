저희 내부 문서 일부 발췌:

Quorum

Quorum is a behavioral mechanic in the cluster to determine operation. Quorum determines the number of server (or "node") failures that the overall cluster can sustain. If half or more of the votes in the cluster (each node having a single vote) become unavailable then the cluster no longer functions. This is to prevent a situation called "split brain" where multiple nodes in a cluster can maintain quorum separate from one another, allowing for unique resource allocation collision where multiple environments are unknowingly accessing unique resources at the same time.

Quorum Disk

Quorum Disk (qdisk) is used as a voting(tiebreaker) mechanism in the event of a loss of communication as well as a communication health check (heuristic ping). This will allow the cluster to operate if either both nodes are online, or a single node & the heuristics.

확인 결과, 해당 기능은 7.3 부터 tech preview 형식으로 제공되었습니다.

Support for quorum devices in a Pacemaker cluster
Starting with Red Hat Enterprise Linux 7.3, you can configure a separate quorum device (QDevice) which acts as a third-party arbitration device for the cluster. This functionality is provided as a Technology Preview, and its primary use is to allow a cluster to sustain more node failures than standard quorum rules allow. A quorum device is recommended for clusters with an even number of nodes and highly recommended for two-node clusters. For information on configuring a quorum device, see https://access.redhat.com/documentation/en-US/Red_Hat_Enterprise_Linux/7/html/High_Availability_Add-On_Reference/. (BZ#1158805)

다만, 앞서 말씀드린바와 같이 7.4 부터 정식 지원됩니다.
