문의 주신 내용에 대해 답변 드리겠습니다.

The Foreshadow / L1-terminal-fault attack were assigned the following CVE numbers:
CVE-2018-3615 for attacking SGX(Intel Software Guard Extensions)
CVE-2018-3620 for attacking the OS Kernel and SMM mode. (operating systems and System Management Mode (SMM) running on Intel processors)
CVE-2018-3646 for attacking virtual machines. (virtualization software and Virtual Machine Monitors (VMM) running on Intel processors)

위 CVE-2018-3620, CVE-2018-3646에 대한 패치로 인해 적용되는 항목들
1) PTE Inversion : 커널소스코드에 하드코딩 적용되어 disabled 불가
2) L1D flush : 조정 가능 (never / conditional / always)
3) Hyperthreads : 조정 가능 (disable / enable)

1) 커널업데이트를 진행하면 성능저하나 다른 이슈나 고려해야 할 사항이 있을까요?
- 베어메탈 상의 OS : PTE Invesion 적용에 영향을 받으며, disable은 불가합니다.
가상 환경을 사용하지 않는 베어메탈 환경에서는 L1TF에 대한 성능 영향 이슈는 미미합니다.
추가적인 고려사항으로 해당 커널에 함께 포함된 Meltdown/Spectre 패치에 대한 성능 이슈도 확인해 보셔야 합니다.

- VM Guest OS : 하이퍼바이저에서 L1TF 취약점을 어떻게 조치하는 것에 따라 성능 영향도가 달라집니다.

2) 커널과 함께 microcode도 업데이트가 필요한가요?
- OS 패치 외에도 microcode를 업데이트도 필요합니다. 관련해서는 H/W 벤더의 문의도 필요합니다.

>> 아래 내용 참조
해당 취약점은 운영 체제의 취약점은 CVE-2018-3620가 할당했으며, 가상화의 취약점은 CVE-2018-3646이 할당되었습니다.
이 문제는 업계에서는 L1 Terminal Fault (L1TF)라고하며 보안 관계자들 사이에서는 "Foreshadow"라고 되어 있습니다.

L1 Terminal Fault의 취약점을 이용하여 악의적인 사용자는 일반적으로 운영 체제 및 하이퍼 바이저에서 강제 및 관리되는 메모리 액세스의 보안 제어를 우회 할 수 있습니다.
일반적으로 운영 체제 및 하이퍼 바이저에 의해 관리되는 page tables는 응용 프로그램, 운영 체제의 커널 자체 및 게스트 가상 머신 인스턴스가 액세스할 수 있는 메모리의 위치에 대한 정보를 프로세서에 제공합니다.
이러한 page table은 유효성을 나타내는 현재 비트가 포함된 PTE로 구성이 되며, 공격자는 L1TF를 이용하여 유효한 PTE를 인식하는 intel 프로세서 로직을 악용하여, 프로세서 L1 데이터 캐시에 캐시 되어 있는 물리적 메모리 위치를 접근 가능합니다.
첫 번째는 Intel SGX 에만 영향을 커널과 마이크로 코드 업데이트를 실시하는 것으로 취약점에 대해서 해결이 가능합니다. 두가지는 운영 체제 및 하이퍼 바이저에 의해 실행되는 소프트웨어 수준의 업데이트가 필요하며, 아래의 해결 탭 부분을 참조하시기 바랍니다.
자세한 보안 취약점 설명에 대해서는 아래 페이지를 방문 후 배경정보(Background Information) 확인을 부탁드립니다.

1) 심각도 등급 : 중요 (Important)


2) 영향 조건
- 영향에 관련 패키지 업데이트
- https://access.redhat.com/security/vulnerabilities/L1TF' 에 방문 후, Diagnose(진단)에 있는 detection script(진단 스크립트)인 cve-2018-3620--2018-08-14-1255.sh 사용하여
현재 사용하시는 시스템에서 확인이 가능합니다.

- 현재 Red Hat에서는 L1TF 취약점을 해결하기 위해, Ansible playbook을 제공하고 있습니다.
* 첫 번째 CVE-2018-3620-fix_disable_ht.yml은 시스템을 다시 시작하지 않고 시스템의 하이퍼 스레딩을 비활성화를 통해 L1TF를 해결하도록 업데이트되지 않은 시스템에 적용이 가능 합니다.
현재의 플레이북은 시스템이 하이퍼바이저로 동작하고, 실행중인 VM이 있는지 확인을 합니다. 하지만 하이퍼 쓰레딩 기능을 비활성화로 설정하면, 이전에 설정된 혹은 현재 설정한 특정 코어/쓰레드의 cpu 선호도를 가진 응용프로그램에서
예기치 않는 문제가 발생할수 있습니다. 시스템의 커널에 필요한 기능이 포함하지 않다면 해당 기능에 대한 메세지와 함께 플레이북은 실패 메세지를 보여주게 됩니다.
* 두번째 CVE-2018-3620-apply_settings.yml, L1TF의 취약점을 해결된 커널로 업데이트 한후, L1TF에 대한 커널 기능을 활성화 시키는 플레이북입니다.
자세한 사용법은 Diagnose에 있는 ansible-palybook에 대한 사용법은 참조 부탁드립니다.

3) 각 버젼 별 패키지 목록
- 관련 패키지 목록에 대해서는 'Resolve' 탭을 참조 해주시길 바랍니다.
https://access.redhat.com/security/vulnerabilities/L1TF
해당 패치는 시스템의 Downtime을 요구하는 작업이며, 해당 보안 취약점은 로컬에 인가되지 않는 공격자에 의한 보안취약점이며, 서비스에 영향이 없도록 업데이트 일정을 계획하시어 해당 패치 버전으로 업데이트를 권장 드립니다.
