CVE 보안최약점이 조치되었는지 확인요청드립니다.

A) 아래의 도표는 RHEL 6.0 부터 6.9 가 릴리즈된 날짜입니다.

그래서, 아래와 같이 현재 RHEL 6.0 에서 RHEL 6.9 가 설치된 머신들은 해당 CVE 보안패치가 release 된 이후의 날짜에
release 가 된 RHEL 버전으로 업데이트 하셔야 할것입니다.

[1] https://access.redhat.com/security/cve/cve-2011-3192
- RHEL 6.0 과 6.1 이 설치되어 기동중인 머신들은 반드시 RHEL 6.2 이상으로 업데이트 해야 합니다.

[2] https://access.redhat.com/security/cve/cve-2011-3368
- RHEL 6.0 과 6.1 이 설치되어 기동중인 머신들은 반드시 RHEL 6.2 이상으로 업데이트 해야 합니다.

[3] https://access.redhat.com/security/cve/cve-2013-1862
- 현재 RHEL 6.0 에서 6.4 버전들이 설치되어서 기동중인 머신들은 모두 6.5 이상으로 업데이트 해야 합니다.

[4] https://access.redhat.com/security/cve/cve-2014-0226
- 현재 RHEL 6.0 에서 6.5 버전들이 설치되어 기동중인 머신들은 모두 RHEL 6.6 이상으로 업데이트 해야 합니다.

[5] https://access.redhat.com/security/cve/cve-2017-7679
- 이 이슈는 어떠한 RHEL 6.x 에 해당이 되지 않고, 최신 이슈이기 문에 이전 안내와 같이 단지 httod 패키지만 업데이트
해야합니다.

httpd 패키지 버전을 최신으로 업데이트하거나 httpd-2.2.15-60.el6_9.5.x86_64.rpm 를 설치하시면 6.0~6.9 버전 모두에 대해서 번들 http에 대한 상기 CVE 보안취약점을 해결 할 수 있습니다. 최신 버전 상태로 유지해주시기 바랍니다.
