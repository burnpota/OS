안녕하세요,

> Apache 2.4에서 Directory permission 설정을 Aapche 2.2설정값을 사용해도 되는건지요?

mod_access_compat 모듈이 로드되어 있다면 old access control configuration ("Allow", "Deny" in httpd 2.2)이 여전히 사용 가능합니다. 그러나 httpd 2.4의 새로운 access control configuration "Require" 사용을 권장됩니다.

자세한 내용은 Apache 문서의 링크 [1]에서 아래 내용을 참조하시기 바랍니다:
~~~
Access control

In 2.2, access control based on client hostname, IP address, and other characteristics of client requests was done using the directives Order, Allow, Deny, and Satisfy.

In 2.4, such access control is done in the same way as other authorization checks, using the new module mod_authz_host. The old access control idioms should be replaced by the new authentication mechanisms, although for compatibility with old configurations, the new module mod_access_compat is provided.

Mixing old and new directives
Mixing old directives like Order, Allow or Deny with new ones like Require is technically possible but discouraged. mod_access_compat was created to support configurations containing only old directives to facilitate the 2.4 upgrade.
~~~

감사합니다.

[1] http://httpd.apache.org/docs/2.4/en/upgrading.html#run-time

<Directory /supp1/webdocs/mobile_domain/wmobile>

You also need to the similar configuration for mobile_domain.httpd.conf, too.

안녕하세요

현재 문의하신 4가지 에러 내용에 관해 자세히 살펴보는 중입니다만, "AH01797: client denied by server configuration" 에러를 제외한 나머지 에러들은 Apache 자체가 아닌 WebLogic Server Proxy Plug-In 에러들로 판단됩니다.

에러 메세지 WRITE_ERROR_TO_CLIENT, READ_ERROR_FROM_CLIENT, NO_RESOURCES 를 포함하는 에러들과 소스 코드로 보이는 ApacheProxy.cpp, BaseProxy.cpp, ap_proxy 들은 Apache의 소스코드가 아닙니다.
해당 에러의 내용을 Red Hat 공식 문서 또는 Knowledge Base solution article, JIRA 에서 찾을 수 없습니다. 다만 웹검색을 통해서 WebLogic Server Proxy Plug-In 의 소스 코드이자 에러 로그들로 판단됩니다.

예를 들어,
~~~
<3484015258139382> *******Exception type [NO_RESOURCES] (apr_socket_connect call failed with error=70007, host=100.5.64.27, port=17600 ) raised at line 1682 of URL.cpp
Trying POST /ib20/act/FIL0000000000063 at backend host '100.5.64.27/17600; got exception 'NO_RESOURCES: [os error=115, line 1682 of URL.cpp]: apr_socket_connect call failed with error=70007, host=100.5.64.27, port=17600 '
~~~
위 에러에 관해서는 Oracle WebLogic Server Proxy Plug-Ins 문서[1]에서 상세한 내용을 확인 할 수 있습니다.
~~~
weblogic: *******Exception type [NO_RESOURCES] (apr_socket_connect call failed with error=70007, host=xx.xx.xx.xx, port=xxxx) raised at line xxxx of URL.cpp

This usually occurs if WLS server is too busy to respond to the connect request from the WebLogic Proxy Plug-In. This can be resolved by setting WLSocketTimeoutSecs to a higher value. This allows the WebLogic Proxy Plug-In to wait longer for the connect request to be responded by the WLS server.
~~~

"AH01797: client denied by server configuration" 에러를 제외한 나머지 3개의 에러에 관해서는 Oracle 에 문의하시길 바랍니다.

"AH01797: client denied by server configuration" 에러에 관해서는 Red Hat Knowledgebase article [2]에 따르면 Allow and Deny directives 설정이 필요해보입니다. [3]

이에 관한 정확한 가이드는 글로벌 Red Hat Web Servers speciality group 에 도움이 필요하여 문의 후 답변이 확인이 되는데로 업데이트 하도록 하겠습니다.

하루 정도 시간이 걸릴 수 있는 점 양해부탁드립니다.
