안녕하세요,

Red Hat Global Support Service 를 이용해주셔서 감사합니다.

Mar 11 07:10:46 FLPTRSD01 multipathd: sdc: hds prio: SCSI error
Mar 11 07:10:46 FLPTRSD01 multipathd: ext0: load table [0 1048576000 multipath 0 0 2 2 round-robin 0 1 1 8:32 1 round-robin 0 1 1 8:48 1]
Mar 11 07:11:07 FLPTRSD01 multipathd: ext0: load table [0 1048576000 multipath 0 0 2 1 round-robin 0 1 1 8:48 1 round-robin 0 1 1 8:32 1]


올려주신 위의 에러 메시지는 multipath 의 priority callout 프로그램을 통해서 logging 이 된것이고, 이 priority 프로그램은
이 path 의 priority 를 결정하기 위해서 SG_IO ioctl 을 보내게 되는데, SCSI error 는 SG_IO IOCTL 의 결과가 SG_INFO_OK
가 아닐때 ( 비정상적일때 ) logging 이 된것입니다.


그래서, 스토리지 벤더사로 해당 이슈에 대해서 기술문의 하신 뒤에 혹시 저희 Red Hat 으로부터 지원이 필요하시다면 피드백 해주시길
부탁드립니다.

위의 로그 메시지에 대한 상세한 내용은 아래의 참고문서[1] 에서 보실수 있습니다.


[ 참고문서 ]

[1] multipathd daemon logging "prio: SCSI error" messages
- https://access.redhat.com/solutions/1335913
