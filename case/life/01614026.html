안녕하세요!
삼성생명 RHEL6 번들 apache 사용 중 SSLCipherSuite 보안취약점이 발견되어 해당 설정 변경 후 하기의 에러로그가 발생하며 정상 기동이 되지 않아 하기와 같이 질문드립니다.

에러로그 스택
### 중략 ###
22527 [Tue Apr 05 20:29:51 2016] [error] Unable to configure permitted SSL ciphers
22528 [Tue Apr 05 20:29:51 2016] [error] SSL Library Error: 336646329 error:1410D0B9:SSL routines:SSL_CTX_set_cipher_list:no cipher match
22529 [Tue Apr 05 20:30:25 2016] [error] Unable to configure permitted SSL ciphers
22530 [Tue Apr 05 20:30:25 2016] [error] SSL Library Error: 336646329 error:1410D0B9:SSL routines:SSL_CTX_set_cipher_list:no cipher match
### 중략 ###

1. 자사 보안팀의 권장사항
ERR_SSL_VERSION_OR_CIPHER_MISMATCH 문제가 발생하며
접근이 불가한 내역들이 서비스 데스크 통해서 다수 접수 되고 있어 SSL 에 대한 설정 변경이 조속히 필요함.
더 이상 안전하지 않다고 간주되는 RC4 에 대한 설정 변경 필요.

2. httpd-ssl.conf 작업 내역
AS_IS)

SSLProtocol all -SSLv2
SSLCipherSuite ECDHE-RSA-AES256-SHA256:AES128-GCM-SHA256:RC4:!LOW:!aNULL:!EDH:!MD5:!SSLv2

TO_BE)
SSLProtocol all -SSLv2
SSLCipherSuite ECDHE-RSA-AES256-SHA256:AES128-GCM-SHA256:!RC4:!LOW:!aNULL:!EDH:!MD5:!SSLv2
c. Apache 재가동

해당 로그로 검색 시 ( SSL Library Error: 336646329 ) 유사 케이스로 https://access.redhat.com/solutions/1312733 가 검색되는데 확인 요청드립니다.

감사합니다.
