안녕하세요,

"Unsupported major.minor version xx" 에러는 높은 버전의 JVM에 의해 컴파일된 class 파일을 낮은 버전은 JVM을 통하여 read 할때 발생합니다. [1]

JBoss Enterprise Application Platform (EAP) 7 Supported Configurations [2]를 보시게 되면, EAP7은 1.8 JVM에 대해 supported 되어 있습니다.

"Jboss7.0_설치시_오류"에 보면 아래와 같은 환경 변수들이 보입니다.
~~
SYSTEM_java_library_path = /usr/java/packages/lib/amd64:/usr/lib64:/lib64:/lib:/usr/lib
eap.native.parent.path = jboss-eap-6.4
~~

env 명령어를 통해 혹시 user 환경에 jdk7이나 eap6.4와 관련된 설정이 있는지 확인 부탁드립니다.

Download 파일은 아래 사이트의 "Red Hat JBoss Enterprise Application Platform 7.0.0 Installer" 를 받으신 건 맞으시죠?
https://access.redhat.com/jbossnetwork/restricted/listSoftware.html?product=appplatform&downloadType=distributions

참고로, 로컬환경에서 Oracle JDK 1.8을 가지고 위 링크의 jar 파일을 이용한 install은 정상적으로 수행되고 있습니다.

또한 [2]에서 확인해 보시면 아시겠지만, CentOS의 경우 Red Hat에서 테스트된(검증된) OS가 아닙니다.
따라서 CentOS에서 발생되는 문제에 대해서는 기술지원이 힘들다는 것을 숙지하여 주시기 바랍니다.

내부 테스트용이라 하시면, 향후 삼성생명에서 EAP7으로의 upgrade가 계획중이신 것인가요?


[1] https://access.redhat.com/solutions/2152781
[2] https://access.redhat.com/articles/2026253
