>OpenSSL FREAK 취약점 관련해서 아래와 같이 보안 패치 공지를 받았는데요

최든 OpenSSL 보안취약점은 FREAK가 아닌 DROWN입니다. 업무에 참고하시기 바랍니다.
======================
>open ssl은 linux 에 기본적으로 설치되어있는 패키지인데, 그럼 전체 linux 대상으로 해당 패키지를 패치해야하는것인지

OpenSSL은 말씀하신 것 처럼 기본적으로 설치되어 있는 패키지입니다. 우선 Red Hat의 권장사항은 보안이슈인 만큼 모든 시스템을 대상으로 패치를 하는 것을 권장 합니다.
======================
>open ssl서비스를 사용하는 대상에 대해서만 패치 해도 되는것인지 확인 부탁드립니다.

보안 패치인 만큼 모든 시스템을 패치하는 것을 권장합니다. OpenSSL 서비스만 사용하는 대상에 대해서만 패치를 하시는 것은 고객사에서 결정하실 부분으로 저희가 결정해드릴 수 있는 부분은 아닌 것 같습니다.

다만 첨언을 드린다면 OpenSSL을 사용하지 않는다면 보안취약 사항을 피할 수 있는 잆습니다. 다만 현재 OpenSSL에서 SSL/TLS를 사용은 HTTPD 등 다양한 곳에서 이용을 하소 있음에 따라, 해당 부분의 사용여부를 정확하게 분리할 수 있다면 해당 패치를 조금 늦게 하더라도 큰 위험이 되지는 않습니다.


결론적으로 OpenSSL을 사용하지 않음을 정확하게 확인하실 수 있다면, 사용하는 시스템을 우선적으로 패치를 진행하시고 사용하지 않은 시스템들은 향후 시스템 리부팅이 있는 시점에 패치를 반영하는 것도 하나의 방안이 될 수는 있을 것 같습니다.

업무에 참고하시기 바랍니다.

안녕하세요,
Red Hat Technical Account Manager 한진구 입니다.

1일 공개된 CVE-2016-0800 관련 안내입니다.
리눅스 시스템 전반적으로 사용하는 SSLv2를 이용한 TLS에 대한 프로토콜에 대한 취약점이며 신속하게 대응하실 수 있도록 사전 안내드립니다.


* CVE 버전 : https://access.redhat.com/security/cve/cve-2016-0800
- 대상 버전 : Red Hat Enterprise Linux 5,6,7 및 파생 제품
- 관련 패키지 : openssl
- 취약점 내용 :


* 설명
SSLv2/TLS를 사용하는 보안 채널에서 외부/내부 공격자가 한개의 포트에 여러 SSLv2을 사용한 공격을 통해 서버와 클라이언트 사이에서 사용 되는 RSA 기 반의 개인 키를 취득할수 있는 취약점이 발견 되었습니다. 이러한 새로운 공 격 형태를 Decrypting RSA with Obsolete and Weakened eNcryption 명하며, DROWN 이라고 지칭 합니다.


* 영 향 도

1) 심각도 등급 : 중요 (Important)

2) 영향 조건
- SSLv2를 이용한 TLS를 사용하는 환경.
- https://access.redhat.com/labs/drown/ 를 통해서 DROWN-test.sh 확인이 가능.
- https://drownattack.com/#check 웹을 통해서 사이트 주소 입력 후 취약점 이 확인 가능.


* 해소 방안
- 제품별 openssl 패키지 업데이트(아래참고)

1) RHEL 5
- openssl-0.9.8e-39.el5
- openssl-perl-0.9.8e-39.el5
- https://access.redhat.com/errata/RHSA-2016:0302.html

3) RHEL 6
- openssl-1.0.1e-42.el6_7.4
- openssl-perl-1.0.1e-42.el6_7.4
- openssl-static-1.0.1e-42.el6_7.4
- https://rhn.redhat.com/errata/RHSA-2016-0301.html

4) RHEL 7
- openssl-1.0.1e-51
- openssl-libs-1.0.1e-51
- openssl-perl-1.0.1e-51
- openssl-static-1.0.1e-51
- https://rhn.redhat.com/errata/RHSA-2016-0301.html

다수의 시스템을 운영하는 경우, Downtime이 없어야 하는 미션 크리티컬 시스 템을 운영하는 경우, 업데이트와 재부팅이 어려움이 있다는 것 알고 있습니 다. 다만 중요한 보안 취약점이고 언론을 포함 널리 알려진 사안인만큼 적극 적인 대응이 필요할 것으로 생각됩니다.

감사합니다.
