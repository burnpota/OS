안녕하세요,

Red Hat Global Support Service를 이용해주셔서 감사합니다.

해당 메세지는 케이스에 직접 올려주신 KBase 에 기술되어 있듯이
일반적으로 하드웨어 장치에 의존하는 메세지입니다.
따라서, 하드웨어적 조사의 선행이 권장됩니다.
기존 유사 사례 등에서도 HBA 교환 등으로 해결된 사례 등이 보고되어 있습니다.

OS 차원에서 추가적으로 조사하고 싶으시다면 해당 KBase 에 기술되어 있듯이
qla2xxx 드라이버 레벨에서 로깅레벨을 조정하여 확장 로그를 기록시킬 수 있습니다.
다만, 메세지 파일을 QLogic 에 의해 제어되는 디스크로부터 로컬 디스크로 이동하거나,
네트워크 경우로 리모트 로그지점으로 이동시키는 것이 권장됩니다.

$ chmod u+w /sys/module/qla2xxx/parameters/ql2xextended_error_logging
$ echo "1" > /sys/module/qla2xxx/parameters/ql2xextended_error_logging
$ sysctl -w dev.scsi.logging_level=0x1003

상기 내용은 직접 남겨주신 다음 KBase 를 참조하실 수 있습니다.

* "Abort command Issued" messages appear in /var/log/messages file
https://access.redhat.com/solutions/27624

감사합니다.
