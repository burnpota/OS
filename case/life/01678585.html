안녕하세요

Red Hat Global Support Services 를 이용해 주셔서 감사합니다.

https://access.redhat.com/security/cve/cve-2013-5704

A flaw was found in the way httpd handled HTTP Trailer headers when processing requests using chunked encoding. A malicious client could use Trailer headers to set additional HTTP headers after header processing was performed by other modules. This could, for example, lead to a bypass of header restrictions defined with mod_headers.

mod_headers 모듈에 HTTP 트레일러가 요청을 처리하는 중에 HTTP 헤더를 교체하도록 허용하는 에러가 존재한다. 이로 인해 공격자는 임의의 헤더를 주입할 수 있으며, 또한 다른 모듈을 부적절하게 작동하게 할 수 있다. (CVE-2013-5704)

요청드린대로 고객사의 취약점검 툴에 어떤 CVE 넘버에 의한 취약점이 검출되었는지 확인이 필요합니다.

안녕하세요
자사 보안팀의 보안취약점에 header 관련하여 보완을 권장받아 하기와 같이 적용을 하였습니다.

httpd.conf

## 중략 ##

<ifModule mod_headers.c>
Header set X-XSS-Protection "1; mode=block"
Header always append X-Frame-Options SAMEORIGIN
Header set X-Content-Type-Options: "nosniff"
# Header set Content-Security-Policy "default-src 'self' 'unsafe-inline'; script-src 'self' 'unsafe-inline'; object-src 'self'"
Header set Content-Security-Policy "script-src 'self'; object-src 'self'"
Header always set Strict-Transport-Security "max-age=31536000"
</ifModule>

##이하 생략 ##

고객사에서 사용 중인 httpd server의 정보는 하기와 같습니다.
RHEL 7.2 : apache 2.4.6
RHEL 6.4 : apache 2.2.15 (httpd-2.2.15-39.el6)

apache 2.4.x 버젼에서는 적용 후 보안팀 확인결과 정상적으로 보완됨을 확인하였는데 2.2.x 버젼은 여전히 취약하다고 나옵니다.

2.2.x용 보안가이드를 요청드립니다.
