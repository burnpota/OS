안녕하세요,

Red Hat Global Support Service 를 이용해주셔서 감사합니다.

volume_list — This acts as a filter through which all requests to activate a logical volume on this machine are passed.
A logical volume is only activated if it matches an item in the list.

위와 같이 기술된 man 페이지에서 보실수 있듯이, volume_list 에 정의되지 item 으로 매치가 되어야 activation 이 가능합니다.
따라서, 이전의 작업 history 가 어떠했는지는 정확히 알수 없는 상태이긴 하나, 현재의 이슈는 misconfiguration 이슈로 예상
되므로, volume_list 내에 부팅시 activate 가 실패된 vg/lv 를 아래와 같이 기입하신 뒤에, initramfs 를 백업 및 재생성
이후 리부팅시 정상적으로 해당 볼륨들이 활성화 되는지 확인해보시길 바랍니다.


1) volume_list 추가작업 ( VG01 은 예제이므로, 정확한 vg 이름을 입력하십시오. )

$ vi /etc/lvm/lvm.conf
~~ snip
volume_list = [ "VG01", "@RTSWEB_1" ]
~~ snip

2) initramfs 백업
$ cp /boot/initramfs-$(uname -r).img /boot/initramfs-$(uname -r).img.$(date +%m-%d-%H%M%S).bak

3) 새 initramfs 리빌드
$ dracut -f -v

4) reboot


감사합니다.
