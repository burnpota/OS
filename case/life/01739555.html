OPTIONS를 통해 사용 중인 HTTP 메소드 정보를 수집하고 HEAD, PUT, DELETE와 같은 메소드를 사용하여 서비스 취약점 확인, 파일업로드 등의 공격이 가능한 취약점입니다.

유닉스 시스템의 경우 web.xml의 WebDAV(웹 기반 분산형 저작 및 버전관리) 서블릿을 추가하여 PROPFIND, DELETE와 같은 메소드를 사용할 수 있고, 이러한 메소드를 사용하여 원격에서 웹 서버 파일관리(검색, 이동, 수정, 삭제 등)가 가능합니다.


OPTIONS 메소드 활성화되는 경우 allow 된 메소드 확인이 가능한 문제가 있어 보안상 비활성화하는 것을 권고하고 있습니다.

-----------------------------------------------------------------------------
telnet localhost 80
Trying ::1...
Connected to localhost.
Escape character is '^]'.
OPTIONS / http/1.0

HTTP/1.1 200 OK
Date: Wed, 16 Nov 2016 03:39:24 GMT
Server: Apache/2.2.15 (Red Hat)
Allow: GET,HEAD,POST,OPTIONS,TRACE
Content-Length: 0
Connection: close
Content-Type: httpd/unix-directory

Connection closed by foreign host.
-----------------------------------------------------------------------------

OPTIONS 메소드를 추가하게되면 보안 취약점 분석 결과에서 "부적절한 메소드 사용" 이라는 항목이 표시될 수 있을 것으로 보입니다.

따라서, 해당 설정으로 GET POST 이외의 메소드는 활성화하지 않을 것으로 보이며, ssl 을 사용중이거나 가상호스트를 사용중이라면 httpd-ssl.conf 에서도 동일하게 설정할 필요가 있을 것으로 보입니다.

<Directory />
<LimitExcept GET POST>
Order allow,deny
Deny from all
</LimitExcept>

</Directory>

답변이 늦어 죄송합니다.

다음은 기본 httpd.conf 에 설정할 수 있는 설정입니다.

#
# Control access to UserDir directories. The following is an example
# for a site where these directories are restricted to read-only.
#
#<Directory /home/*/public_html>
# AllowOverride FileInfo AuthConfig Limit
# Options MultiViews Indexes SymLinksIfOwnerMatch IncludesNoExec
# <Limit GET POST OPTIONS>
# Order allow,deny
# Allow from all
# </Limit>
# <LimitExcept GET POST OPTIONS>
# Order deny,allow
# Deny from all
# </LimitExcept>
#</Directory>

아래는 고객께서 설정한 부분입니다.
또한, 본 설정을 통해서 GET POST 이외의 method 는 비활성화하는 것으로 알고 있습니다.

<Directory />
<LimitExcept GET POST>
Order allow,deny
Deny from all
</LimitExcept>
</Directory>

해당 내용을 아래와 같이 변경하여 동일한 현상이 검출되는지 확인해주시기 바랍니다.

<Directory />
AllowOverride FileInfo AuthConfig Limit
Options MultiViews Indexes SymLinksIfOwnerMatch IncludesNoExec
<LimitExcept GET POST OPTIONS>
Order allow,deny
Deny from all
</LimitExcept>
</Directory>

취약점 점검 결과에 혹시 CVE 번호가 있는지도 확인해주시면 추가정보를 찾아볼 수 있을 것 같습니다.
