사설 Repo 서버의 시스템이 RHSM에 register 되어 있어야만 reposync 라는 커멘드로 동기화를 진행할수 있습니다.
따라서 먼저 서버를 RHSM에 register를 진행하신후 아래의 절차대로 reposync 후 로컬 repo를 구축하실수 있습니다.

httpd 기반의 repo 서버를 구축하는 방법을 아래와 같이 안내해드리겠습니다.
================================================================
1. httpd 패키지를 설치합니다.
# yum install httpd

2. httpd의 document root에 repo를 생성할 디렉토리를 만듭니다.
# mkdir -p /var/www/html/repo

3. 관련패키지를 설치합니다.
# yum install yum-utils createrepo

4. 동기화하고 싶은 repo를 아래와 같은 명령으로 동기화합니다.<REPOID>는 "# yum repolist"커멘드로 찾을수 있습니다.
# reposync -p /var/www/html/repo -r <REPOID> -l

5. 로컬 repo를 생성합니다.
# createrepo /var/www/html/repo

6. /etc/httpd/conf/httpd.conf 파일에서 아래와 같이 설정합니다.
# vim /etc/httpd/conf/httpd.conf
ServerName <ip_address> <------ repo서버의 IP로 지정합니다.

7. httpd 서비스를 재기동합니다.(서비스를 재기동하기전에 iptables 에서 80 포트를 열어주실 필요가 있습니다.)
# chcon -R -t httpd_sys_content_t /var/www/html/repo/
# service httpd restart
==========================================================
repo서버를 구축하신후 cliet 서버에서 /etc/yum.repo.d/remote.repo파일에서 아래와 같이 설정합니다.
# vim /etc/yum.repo.d/remote.repo

[repo]
name=repo
baseurl=http://<ip_address>/repo
enabled=1
gpgcheck=0

위의 내용은 아래의 기술문서를 참조하실수 있습니다.

TCP connection이 맺어진후 retransmission 횟수에 관련된 컨널 파라메터는
net.ipv4.tcp_retries2 이며, 기본값은 15를 가지게 됩니다.
해당 값이 15라면 TCP connection이 맺어진후 원격서버에서 응답이 없으면,
retransmission을 16번 시도하게 됩니다.

retransmit timeout = (1+2+4+8+16+32+64+128+256+512+600+600+600+600+600+600)*0.2 = 924s
