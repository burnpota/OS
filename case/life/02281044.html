안녕하세요

SDS 김지민 선임입니다.

vg의 여유가 남은 상황에서 lvextend 수행 시 lvol 사이즈 관련 문의드리려고 합니다.

아래와 같이 LLPTSSAPPVG에서 50G의 APP lvol을 삭제하고
70G의 여유 확인 후 LLPTSSAPPVG에 있는 LOG lvol을 lvextend(100%)를 수행하였습니다.

근데 해당 lvol이 70GB까지 밖에 증가가 안되어서 문의드리려고 합니다.

root@SLPTSSAL05 / # lvs
LV VG Attr LSize Pool Origin Data% Meta% Move Log Cpy%Sync Convert
APP LLPTSSAPPVG -wi-a----- 50.00g
LOG LLPTSSAPPVG -wi-a----- 30.00g
APP LRPTSSAPPVG -wi-ao---- 50.00g
sysadmin vg0 -wi-ao---- 6.99g
CRASH vg9 -wi-ao---- 114.00g
DUMP vg9 -wi-a----- 111.00g
root@SLPTSSAL05 / # lvremove /dev/LLPTSSAPPVG
Do you really want to remove active logical volume LLPTSSAPPVG/APP? [y/n]: n
Logical volume LLPTSSAPPVG/APP not removed.
Do you really want to remove active logical volume LLPTSSAPPVG/LOG? [y/n]: n
Logical volume LLPTSSAPPVG/LOG not removed.
root@SLPTSSAL05 / # lvremove /dev/LLPTSSAPPVG/APP
Do you really want to remove active logical volume LLPTSSAPPVG/APP? [y/n]: y
Logical volume "APP" successfully removed
root@SLPTSSAL05 / # vgs
VG #PV #LV #SN Attr VSize VFree
LLPTSSAPPVG 1 1 0 wz--n- 100.00g 70.00g
LRPTSSAPPVG 1 1 0 wz--n- 50.00g 0
vg0 1 1 0 wz--n- 6.99g 0
vg9 1 2 0 wz--n- 225.00g 0
root@SLPTSSAL05 / # lvextend -l 100%free /dev/LLPTSSAPPVG/LOG
Size of logical volume LLPTSSAPPVG/LOG changed from 30.00 GiB (7680 extents) to 70.00 GiB (17919 extents).
Logical volume LLPTSSAPPVG/LOG successfully resized.
root@SLPTSSAL05 / #
root@SLPTSSAL05 / # lvs
LV VG Attr LSize Pool Origin Data% Meta% Move Log Cpy%Sync Convert
LOG LLPTSSAPPVG -wi-a----- 70.00g
APP LRPTSSAPPVG -wi-a----- 50.00g
sysadmin vg0 -wi-ao---- 6.99g
CRASH vg9 -wi-ao---- 114.00g
DUMP vg9 -wi-a----- 111.00g
root@SLPTSSAL05 / # vgs

LLPTSSAPPVG 1 1 0 wz--n- 100.00g 30.00g
LRPTSSAPPVG 1 1 0 wz--n- 50.00g 0
