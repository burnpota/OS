Red Hat Global Support Services를 이용 해 주셔서 감사합니다.


RHEL5 버전에서는 TLS v1.1 혹은 v1.2 기능을 제공 하지 않습니다.
현재 RHEL5 버전에서 TLS v1.0 만 제공 하고 있는 것으로 보입니다.
더 상세한 내용은 아래와 같은 링크에서도 확인 가능 하십니다.

TLS support in RHEL5
- https://access.redhat.com/solutions/1609823


[rpm/yum을 통하여 업데이트 원복 방법]

- rpm로 업데이트 후 원복 방법

rpm를 통하여 패키지 업데이트후 원복 하시려면, 현재 사용 중인 rpm 패키지를
모두 uninstall 하시고 다시 설치 하셔야 합니다.

# rpm -e <package1-name> <package2-name> ...
# rpm -ivh <package1-name> <package2-name> ...

- yum로 업데이트 후 원복 방법

RHEL6/7 경우에 yum history 기능을 이용 하여 원복 하실수 있습니다.

# yum history
# yum history undo <num>

RHEL5 일 경우, yum downgrade 명령어를 이용 하여 원복 하실수 있습니다.

# yum downgrade <package-version>

참고 자료:
How to use yum to downgrade or rollback some package updates?
- https://access.redhat.com/solutions/29617


감사합니다.

안녕하세요,

Red Hat Global Support Services를 이용 해 주셔서 감사합니다.


TLSv1.1 & TLSv1.2를 사용 하기 위해 openssl-1.0.1e-15.el6 또한
httpd-2.2.15-39 버전으로 업데이트 하셔야 합니다.

Does the Apache webserver support TLS version 1.1 and 1.2?
- https://access.redhat.com/solutions/65030

만약 시스템이 Red Hat Network 로 등록 되어 있을 경우에 yum 명령어로
업데이트 하여 의존성을 자동으로 해결 할수 있습니다.

아래와 같이 최신버전으로 업데이트 하시거나,
# yum update
버전을 지정 하여 원하신 버전으로 업데이트
# yum update httpd-2.2.15-39.el6


서버가 Red Hat Network로 등록 되지 않았을 경우 패키지를 다운로드
받으셔서 rpm 명령어로 업데이트 하실수 있습니다.

# rpm -Uvh <package-name>

패키지를 다운로드 받는 링크:
https://access.redhat.com/downloads/content/69/ver=/rhel---6/6.8/x86_64/packages

보내 주신 링크에서 언급 한 httpd 버전은 upstream 버전으로 보이며
RHEL에서 탑재된 버전은 2.2.15-xx 로 보입니다.


감사합니다.
