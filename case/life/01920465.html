Q3) swap 파티션을 생성후 blkid 시 uuid 표시가 안되며 uuid 로 마운트가 안됩니다.
A3) 위의 질문을 저희 스토리지 전문그룹으뢰 의뢰를 하여서 답변을 받았는데, blkid cache 파일인 /etc/blkid/blkid.tab 내에 swap 의 UUID 가
정의되어 있는지 확인해보시고, 아래의 low level 측정을 해서 UUID 를 찾은 다음 다시 UUID 로 마운트를 시도해시고 난 다음 피드백을
요청하였습니다.

$ blkid -p /dev/md127p5

그리고, swap 을 삭제하였다가 재생성 한 뒤에 다시 한번 더 UUID 로 마운트를 시도한 결과도 피드백 해주시길 부탁드립니다.

Red Hat Global Support Service 를 이용해주셔서 감사합니다.

Q1) UEFI 로 설치시 Disk type 이 gpt 로 되는건 가요?
A1) BIOS 에서 UEFI 로 설정이 된 경우에는 2.2 TB 크기의 디스크내 파티션 테이블을 을 지원하므로 OS 설치시 인스톨러인 anaconda 가 이를 식별하고
parted 유틸리티로 GPT 파티션으로 설치한것입니다.

Q2) 파티션 타입이 Microsoft basic 로 표시됩니다.
A2) 파티션 타입이 Microsoft basic 으로 보이는것은 RHEL 이나 리눅스 커널내에서는 이러한 파티션 타입을 조정하는 어떤 코드다 도구가 없기 때문에
아마도 이전에 Windows 시스템이 설치가 되어졌던 디스크였던것으로 예상이 됩니다. 이 정보는 시스템에 무해하므로 무시하셔도 되지만, 만약 변
경을 원하신다면 fdisk 나 parted 로 각 파티션의 타입의 변경이 가능합니다.

Q3) swap 파티션을 생성후 blkid 시 uuid 표시가 안되며 uuid 로 마운트가 안됩니다.
A3) 현재 확인중에 있으니 조금만 더 기다려주시길 부탁드립니다.
