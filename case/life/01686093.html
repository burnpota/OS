안녕하세요,

Red Hat Global Support Services 를 이용해 주셔서 감사합니다.

man 페이지를 보시다시피 pam_cracklib는 패스워드 안정성을 위하여 dictionary 단어들을 설정 할수 없게
설계되어 있으며 특수 단어를 배제하는 기능이 없는것으로 확인됩니다.

pam_cracklib - PAM module to check the password against dictionary words

따라서 만일 패스워드 설정시 dictionary 단어들을 사용하게 하시려면 말씀하신 설정을 진행하셔야 됩니다.
이와 같은 설정을 하여 서버에는 영향이 없지만 패스워드 복잡성이 낮아지게 됩니다.
만일 해당 서버가 폐쇠망에 있다면 큰 문제가 되지 않지만 외부 네트워크와 연결되어 있다면
패스워드가 취약하여 안정성이 떨어지게 됩니다.

password requisite pam_cracklib.so try_first_pass retry=3 type= <----------주석처리
password sufficient pam_unix.so sha512 shadow nullok try_first_pass use_authtok <---use_authtok제거
