안녕하세요,

Red Hat Global Support Service 를 이용해주셔서 감사합니다.

xfs signature detected on /dev/vg1/batch_sam_SAM_work at offset 0. Wipe it? [y/n]: y

위의 메세지는 XFS Filesystem Signature 가 해당 디바이스에 존재함을 나타내고, 단순히 해당 디바이스가 이전에
XFS 파일시스템을 사용했다는것을 의미하는 Signature 가 있음을 나타냅니다.

이전에 이 디바이스 상에 XFS 파일시스템이 있었었고, 이 디바이스에 LV 를 만들려고 시도했지만 libblkid 라이브러리는
XFS Log Signature Offset 을 제공하지 않았을 것입니다.

즉, libblkid 는 감지된 Signature 목록을 Return 했지만 각 Signature 에 대한 Offset 과 길이는 리턴하지
않은것입니다.

결국, wipefs command 와 LVM 도구는 해당 Block Device 로부터 XFS 로그를 지우는데 실패해서
"Failed to get offset of the xfs_external_log signature on /dev/vg1/lv1" 에러가 발생한것입니다.

위 에러는 심각한 에러가 아니기에 무시될 수는 있습니다만, 개선을 위해서 Bug 가 수정된 아래의 패키지들을 업데이트 하실
것을 권장드립니다.

libblkid-2.23.2-33.el7.x86_64.rpm
libblkid-devel-2.23.2-33.el7.x86_64.rpm
libmount-2.23.2-33.el7.x86_64.rpm
libmount-devel-2.23.2-33.el7.x86_64.rpm
libuuid-2.23.2-33.el7.x86_64.rpm
libuuid-devel-2.23.2-33.el7.x86_64.rpm
