Q) 하기의 CVE 리스트가 조치된 번들 http server 마이너 버젼이 있는지요? 최신 패치버젼으로 설치진행하면 되는지 확인요청드립니다.
ㄱ. CVE-2013-1862 : Excute Code
ㄴ. CVE-2014-0226 : Excute Code
ㄷ. CVE-2017-7679 : DOS


A) 요청주신 위의 문의내용을 각 CVE 코드에 대응하여 아래와 같이 안내드립니다.

A-1) CVE-2013-1862
이 보안이슈는 2013 년도에 발견이 되어 당시 RHEL 7 이 발매된 이전의 시점이서 RHEL 7 에서는 적용이 된다고 말하기 어렵습니다.
다만, RHEL 7 에 포함이 되어 있는 httpd 버전이 이미 훨씬 상위버전이므로 ( RHEL 6 : httpd-2.2.15 , RHEL 7 : httpd-2.4.6 ) 이
미 해당 패치가 보유가 되었을것입니다. 그래서, 이 이슈를 위해서는 특별히 업데이트를 하지 않으셔도 무방할것입니다.

* 참고문서
[1] https://access.redhat.com/security/cve/cve-2013-1862

A-2) CVE-2014-0226
이 보안 이슈는 이미 httpd-2.4.6-18 버전에서 해소가 되었기 때문에 현재 사용하는 버전에서 fix 된 코드를 포함하고 있어서 별도로
업데이트를 하실 필요가 없습니다.

* 참고문서
[2] https://access.redhat.com/security/cve/cve-2014-0226

A-3) CVE-2017-7679
이 보안 이슈는 RHEL7 의 httpd-2.4.6-67 에서 수정이 되었기 때문에 업데이트가 필요로 합니다.

* 참고문서
[3] https://access.redhat.com/security/cve/cve-2017-7679

* 최신 보안 업데이트를 적용 하기위해서 아래와 같이 yum 명령을 실행하셔서 최신버전의 httpd 로 업데이트 하시거나
아래의 경로[4] 에서 최신버전의 httpd 패키지를 다운로드 받아서 설치하실수도 있습니다.

$ yum update httpd

[4] https://access.redhat.com/downloads/content/69/ver=/rhel---7/7.4/x86_64/packages
