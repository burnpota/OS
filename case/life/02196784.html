====
Updates for Affected Products


Product
Package
Advisory/Update
Red Hat Enterprise Linux 6
kernel
In progress

Mitigation

The flaw can be mitigated by reducing the hard stack limit usable by all users in the system. You can do so by modifying the system-wide limits and restarting the system. One adverse side effect is that limiting the stack may crash some large-stack programs. Fortunately, it is uncommon that an application hits this limit.

vi /etc/limits.conf

* hard stack 30720

A SystemTap script provided in BZ#1624498 does mitigate the provided exploit, but changing the limits system wide is a more comprehensive solution.
====
영향을 받는 제품 업데이트

제품 패키지
권고/업데이트
Red Hat Enterprise Linux 6
kernel
진행 중

완화 방법

본 취약점은 시스템의 모든 사용자가 사용할 수 있는 하드 스택 제한을 줄임으로써 결함을 완화시킬 수 있습니다. 시스템 전체 제한을 변경하고 시스템을 다시 시작하면 됩니다. 스택을 제한하면 일부 대형 스택 프로그램이 충돌하는 문제가 발생할 수 있지만 애플리케이션이 이러한 제한에 도달하는 경우는 극히 드뭅니다.
원본

vi /etc/limits.conf

* hard stack 30720


BZ#1624498에서 제공된 SystemTap 스크립트를 통해 취약점을 완화시킬 수 있지만 시스템 전체 제한을 변경하는 것이 보다 포괄적인 솔루션이 될 수 있습니다.
====
이미 확인하셨겠지만 상기를 토대로 다시한번 말씀드리면,

> 1. 커널 패키지 업데이트하는 방법 6버전은 현재 진행중인가요?

진행 중입니다.

> 2. systemtap을 이용한 완화 방법은 무엇인가요?

버그질라(BZ#1624498) 에서 확인하실 수 있습니다.
링크: https://bugzilla.redhat.com/show_bug.cgi?id=1624498

> 3. limits.conf를 통한 아래와 같이 설정한다면 해당 보안이슈를 해결할수 있는건가요?
> vi /etc/limits.conf
>
> * hard stack 30720

포괄적인 완화책이며 systemtap 방식보다 권장됩니다.

> 4. 커널을 업데이트 하지 않고 해결 할수 있는 방법은 systemtap을 이용하는 방법과 limit.conf에 hard stack을 설정하는 방법 뿐인가요?

네, 그렇습니다.

결론적으로, 해당 취약점과 관련하여 RHEL6 제품에 대해 현단계에서는 limits.conf 수정을 통한 완화책을 권장드리며
취약점 패치 릴리즈후에는 커널 패키지를 업데이트하실 것을 권장드립니다.

====
Mutagen Astronomy는 로컬 사용자 권한 상승 취약점에 대한 코드명입니다.
어플리케이션에서 전달 받은 매개 변수(argument) 를 스택에 넣으려면 create_elf_tables 호출하게 됩니다.
현재 create_elf_talbes에 integer overflow 취약점이 발견 되었으며, root 및 관리자 권한이없는 SUID(또는 권한이있는) 바이너리에
접근 할 수있는 로컬 사용자는이 취약점을 사용하여 시스템에 대한 접근 권한을 상승시킬 수 있습니다.

Mutagen Astronomy - 로컬 권한 상승 - CVE-2018-14634
https://access.redhat.com/ko/security/vulnerabilities/3638731 (국문)
https://access.redhat.com/security/vulnerabilities/mutagen-astronomy (영문)

* CVE버젼 :
- https://access.redhat.com/security/cve/cve-2018-14634

* 대상 버전:
- Red Hat Enterprise Linux Server 6 및 파생 버젼
- Red Hat Enterprise Linux Server 7 및 파생 버젼

* 관련 패키지: kernel

* 영 향 도
1) 심각도 등급 : 중요 (Important)

2) 영향 조건
- 영향에 관련 패키지 업데이트

3) 완화 방법
- 패키지 업데이트 (현재 RHEL6는 진행중 입니다)
- systemtap을 이용한 완화 방법. (시스템의 부가적으로 발생되는 시스템 로드)
- /etc/limits.conf 를 통해 모든 사용자가 사용할 수있는 hard stack 제한을 줄임(대규모 stack를 사용하는 응용 프로그램인 경우, 충돌 가능성)

4) 각 버젼 별 패키지 목록
- 관련 패키지 목록에 대해서는 'Resolve' 탭을 참조 해주시길 바랍니다
https://access.redhat.com/security/vulnerabilities/mutagen-astronomy

해당 패치는 시스템의 Downtime을 요구하는 작업이며, 해당 보안 취약점은 로컬에 인가되지 않는 공격자에 의한 보안취약점이며, 서비스에 영향이 없도록 업데이트 일정을 계획하시어 해당 패치 버전으로 업데이트를 권장 드립니다.
====
