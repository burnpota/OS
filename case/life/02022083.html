>> RHEL7 번들 httpd server와 jbcs의 차이점을 문의드립니다.

기본적으로 버전의 차이가 존재합니다.
* RHEL 7: based on upstream v2.4.6
* Red Hat JBoss Core Services (JBCS)5 Apache HTTP Server: Based on upstream v2.4.23


그리고 가장 결정적인 차이가 Aapche와 JBoss EAP 를 연결하는 mod_jk 같은 connector 지원 여부입니다.
Does Red Hat / JBoss offer support for Apache, mod_jk, mod_proxy, or mod_cluster? https://access.redhat.com/solutions/18495
Apache 버전과 EAP 버전별로 지원가능한 connector 에 차이가 존재합니다. 예를 들어 EAP 7의 경우 아래와 같습니다.
* RHEL7's 2.4.6 + EAP 7 => mod_proxy
* JBCS httpd + EAP 7 => mod_proxy+mod_jk+mod_cluster
Backend Worker 서버로 EAP와 연동하여 사용하실 예정이시라면 사용하실 connector를 고려하여 이 부분을 잘 확인해보시기 바랍니다.

JBCS Apache는 EAP와 연동(integration)을 위한 JBoss용 Apache 제품(JBoss Core Services)입니다. 번들 httpd는 기술 지원 자료 및 문서가 거의 없는 반면, JBCS Apache는 다수의 기술 지원 자료와 문서가 존재합니다.

감사합니다.
