안녕하세요,
Red Hat 한진구 입니다.

OpenSSL FREAK와 관련되어 아래 정리하여 답변 드립니다.


1. FREAK에 대한 CVE 간략 설명

* CVE 버전 : https://access.redhat.com/security/cve/CVE-2015-0204
- 대상 버전 : Red Hat Enterprise Linux 5, 6, 7
- 관련 패키지 : openssl
- 취약점 내용 : FREAK으로도 불리우는 본 취약점은, OpenSSL등 암호화된 통신을 수행할때 OpenSSL client가 자신이 요청하지 않은 RSA key를 Accept할 수 있는 보안 취약점으로 이를 이용하면 man-in-the-middle 공격이 가능해질수 있습니다.

과거 미국 정부가 암호화 기술 수출시에 512 bit RSA key 방식까지만 허용해왔습니다. 512 암호화 방식은 그래서 EXPORT-grade라고 불리우는데, OpenSSL 역시 호환성을 위해 512 bit RSA key를 지원하고 있습니다. 512 암호화 방식은 강도가 약하기 때문에 클라우드 환경등 대량의 컴퓨팅 자원을 이용하면 7시간정도에 깨질 수 있는 수준입니다. 물론 대부분의 client/server등은 1024 bit 이상의 암호화 방식을 사용하기 때문에 문제가 없고, 512 bit key를 사용한다 하더라도 SSL session 하나로부터 정보를 추출하고 해석하기 위해 대량의 컴퓨팅 자원을 투입해야 하기 때문에 심각도 자체는 높지 않습니다.

공격 방법은 다음과 같습니다.
1) 공격자가 서버와 클라이언트 중간에서 SSL 통신을 낚아채는 MITM 공격에 성공한 후,
2) 해당 OpenSSL 취약점을 이용해 암호화 요청을 512 bit인 EXPORT-grade key 방식으로 변경하여 512 bit 방식으로 통신하도록 조작할 수 있게 됩니다.
3) 이후 key를 해킹한 후 유출된 데이터를 복호화해 정보 추출하게 됩니다.
======================
