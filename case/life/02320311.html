DHCP 마스터 서버가 없는 상태에서 DHCP 클라이언트 Enable 설정을 하면
어떤 기준으로 IP Address가 할당되는 것이지 알고 싶습니다.
즉, Non-DHCP Server 환경하에서 DHCP Client 설정된
Linux서버가 임의IP를 할당하는 경우 어떤 IP대역을 할당하는지 알고 싶습니다.

문의하는 사유
- 리눅스 서버가 종종 추가되고 있는데, 이때 주사용 NIC은 BOOTPROTO=none 으로 붙이는데
다른 NIC은 BOOTPROTO=dhcp 로 있는경우가 있음
- 이 때 dhcp server가 없기 때문에 dhcp client에서 임의의 IP를 assign하게 되는 것으로
아는데, 이때 주사용 IP 대역에서 임의 지정되는 것으로 추정됨.
(Windows의 경우는 169.x 대역으로 지정되는 것 같음)
================================================================================

Q) Non-DHCP Server 환경하에서 DHCP Client 설정된 Linux서버가 임의IP를 할당하는 경우 어떤 IP대역을 할당하는지 알고 싶습니다.

A) Linux 서버는 ifcfg-ensXX 장치의 BOOTPROTO=dhcp일 경우 네트워크 장치를 기동 시 broadcast request 를 하고 dhcp가 요청을 받아 IP 정보를 reply 받아 IP를 설정합니다.
그런데 request에 대한 reply를 받지 못 하는 경우(dhcp 서버 미존재 등) 임의의 IP가 아닌 none IP 상태로 장치설정이 됩니다.
즉, Windows OS 처럼 169.XXX.XXX.XXX 와 같은 IP를 세팅하지 않습니다.
