제목 : hostid 가 변경되는 문제

======================
사전문의

어떤 문제/오류/결함이 발생했습니까? 기대하시는 결과는 무엇입니까?

안녕하십니까, hostid 관련 문의가 있어 Case 올립니다.
RHEL 5.6 버젼에서 hostid를 고정적으로 지정할 수 있는 방법이 있는지 알고싶습니다.
hostid가 /etc/hosts에 입력된 내용과 실제 hostname의 결과값이 매칭되어 나오는 것으로 확인되는데 IP가 변경된 이후에 /etc/hosts에 입력내용을 변경하면 무조건 hostid가 변경되는지
혹, /etc/hosts에 입력된 내용에 상관없이 고정적으로 hostid를 부여하는 방법이 있는지 문의드립니다.

안녕하세요,

Red Hat Global Support Service를 이용해주셔서 감사합니다.

문의하신 내용에 대하여 안내를 드리면 다음과 같습니다.

hostid 는 /etc/hostid 에 저장된 값이 있을 경우 가장 우선순위가 높게 참조됩니다.
해당 파일이 존재하지 않을 경우에 한해 IP 어드레스 등을 기반으로 재생성됩니다.
따라서, 기존에 생성된 /etc/hostid 파일을 백업해두시면 해당 값을 유지하실 수 있습니다.

상기에 관련된 KCS 링크는 다음과 같습니다:
How is a hostid generated?
- https://access.redhat.com/solutions/34365
~~~
hostid is the numeric identifier for the host, In the glibc implementation, the hostid is stored in the file /etc/hostid.

In the glibc implementation, if gethostid() cannot open the file containing the host ID, then it obtains the hostname using gethostname(), passes that hostname to gethostbyname_r() in order to obtain the host’s IPv4 address, and returns a value obtained by bit-twiddling the IPv4 address. (This value may not be unique.)
~~~

다음은 업무에 참조하시도록 작업 예를 남겨드립니다.
~~~
예)
[root@localhost ~]# hostid
b4d10fb1
[root@localhost ~]#
[root@localhost ~]# cp -p /etc/hostid /etc/hostid.back
[root@localhost ~]#
[root@localhost ~]# hostid
b4d10fb1
[root@localhost ~]#
[root@localhost ~]# genhostid
[root@localhost ~]#
[root@localhost ~]# hostid
2e03077c
[root@localhost ~]#
[root@localhost ~]# cp -p /etc/hostid.back /etc/hostid
cp: overwrite `/etc/hostid'? y
[root@localhost ~]#
[root@localhost ~]# hostid
b4d10fb1
~~~

감사합니다.
