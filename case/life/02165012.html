시스템 모니터링 관련 아래와 같이 고객 요청 문의 드립니다.

Q1) CPU / MEMORY 실시간 조회
해당 리소스 조회를 위해 sar수집주기를 최소로 맞출 예정입니다.
/etc/cron.d/sysstat 아래 부분 수정을 최소단위로 할 시 수집주기 문의드립니다.

-> sar 수집 주기를 1초 단위로 설정하여도 시스템 부하 및 다른 영향이 없는건가요?
(1~2일정도 수집할 예정 입니다)

A1) 참고문서[1],[2] 에서 설명하는것과 같이 1분 이내로도 주기 설정이 가능하고 SAR 가 서버에 어떤 부하를 주지는 않지만
너무 짧은 주기는 시스템 리소스 사용의 패턴을 모니터링 하기에는 너무 많은 불 필요한 데이터가 쌓일수 있습니다.
1분 이내로 설정도 가능하지만, 저희 자체적으로 최소 테스트 하여 확인한 권장값은 2분입니다.

NOTE: SAR does not add significant load to a server. It safely can be tuned down to 2 minute intervals without seeing a significant problem. SAR also does not grab individual block data.


Q2) 네트워크 I/O 트래픽 조회
네트워크 I/O 허용범위와, 그 트래픽양 조회방법에 대한 문의입니다.
해당부분은 네트워크그룹에서도 모니터링 예정이지만, OS차원에서 고려해야할 부분이 있다면 전달 부탁드립니다.
-> 파일첨부를 통한 스크립트로 모니터링을 진행할 예정입니다.

A2) SAR 데이터가 저장이 되어 있다면 아래의 명령어로 네트워크 상태를 모니터링 할 수 있습니다. - 참고문서[2]
$ sar -n ALL -f /var/log/sa/sa13


Q3) 침해경로
해당 모의훈련 공격 시 소스 IP 혹은 공격자의 정보를 조회가 가능한 지 문의드립니다.
이 부분은 관제그룹과 네트워크에서 자체 에이전트를 통해 조회 예정입니다.
2번과 마찬가지로, 서버 내에 조회가능한 부분이 있다면 간략히 공유 부탁드립니다.
-> OS 에서 조회가 가능하다면 조회 방법 부탁 드립니다.

A3) 위의 질문은 보안이슈와 관련된 질문이어서 저희가 답변을 드리기가 어려운 점이 있습니다만 ( 지원범위 미해당 ),
주로 ftp,ssh,telnet 등과 같은 원격접속의 기록은 /var/log/secure 로그에서 확인이 가능합니다.
