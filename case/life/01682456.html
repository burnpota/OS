안녕하세요
삼성생명의 RHEL6와 RHEL7의 번들 HTTPD server 보안 취약점 관련하여 문의드립니다.

현재 하기의 보안취약점이 발견되었습니다.
"HTTP Strict-Transport-Security" 헤더 누락
"X-XSS-Protection" 헤더가 누락
"X-Content-Type-Options" 헤더가 누락
"Content-Security-Policy" 헤더가 누락

현재 httpd.conf 에 아래와 같이 적용되었는데 2.4.x는 정상 적용되어 보완되었는데 2.2.x는 동일한 설정을 하였음에도 취약하다고 합니다.
혹시 설정값이 다른지 확인을 요청드립니다.

<ifModule mod_headers.c>
Header set X-XSS-Protection "1; mode=block"
Header always append X-Frame-Options SAMEORIGIN
Header set X-Content-Type-Options: "nosniff"
Header set Content-Security-Policy "default-src 'self' 'unsafe-inline'; script-src 'self' 'unsafe-inline'; object-src 'self'"
Header always set Strict-Transport-Security "max-age=31536000"
</ifModule>
