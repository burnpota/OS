안녕하세요,
Red Hat Technical Account Manager 한진구 입니다.

10일 공개된 OpenSSL에 리포트된 다수의 보안 취약점과 관련 하여 사전 안내 드립니다.

* Security Advisory
- https://access.redhat.com/errata/RHSA-2016:0996

1) OpenSSL:ASN.1 encoder에서의 메모리 충돌 보안 이슈(CVE-2016-2108)
- 이것은 2015 년 6 월 11 일에 보안 릴리스가 나온 것과 동일한 보안이슈 관련으로써, d2i_ASN1_TYPE의 ASN.1 parser는 크기가 큰 universal tag에 대해 negative zero로 발생하며, 공격자가 이를 약용해 memory corruption을 유도하거나, 사용자 권한에서 잠재적인 공격 코드를 실행 시킬 수 있는 취약점입니다.

2) OpenSSL:EVP_EncodeUpdate() 함수 에 버퍼 오버플로 (CVE-2016-2105)
- OpenSSL을 사용하는 어플리케이션에서 EVP_EncodeUpdate() 함수를 통해 반환되는 데이터의 크기를 확인하지 않아, 원격 공격자가 이를 약용해 memory corruption을 유도하거나, 사용자 권한에서 잠재적인 공격 코드를 실행 시킬 수 있는 취약점입니다.

3) OpenSSL:EVP_EncryptUpdate()함수 에 버퍼 오버플로 (CVE-2016-2106)
- OpenSSL을 사용하는 어플리케이션에서 EVP_EncryptUpdate() 함수를 통해 반환되는 데이터의 크기를 확인하지 않아, 원격 공격자가 이를 약용해 memory corruption을 유도하거나, 사용자 권한에서 잠재적인 공격 코드를 실행 시킬 수 있는 취약점입니다.

4) OpenSSL:AES-NI CBC MAC 암호화에 따른 응답 판독을 통해 암호화된 트래픽 복호화 보안 이슈(CVE-2016-2107)
- 보안 연결이 AES CBC 암호화를 사용하고 서버가 AES-NI를 지원하는 경우, 중간 지점의 공격자는 응답에 대한 판독을 통해 암호화된 트래픽을 살펴볼 수 있게 됩니다.

5) OpenSSL:BIO_*printf 함수의 메모리 이슈 (CVE-2016-0799)
- OpenSSL을 사용하는 어플리케이션에서 BIO_*printf() 함수에 전달되는 데이터의 크기를 확인하지 않아, 원격 공격자가 이를 약용해 memory corruption을 유도하거나, 사용자 권한에서 잠재적인 공격 코드를 실행 시킬 수 있는 취약점입니다.

6) OpenSSL:doapr_outch 힘수의 메모리 할당 성공 결과를 전달 하지 못한 이슈 (CVE-2016-2842)
- OpenSSL을 사용하는 어플리케이션에서 doapr_outch() 함수를 통해 반환되는 데이터의 크기를 확인하지 않아, 원격 공격자가 이를 약용해 memory corruption을 유도하거나, 사용자 권한에서 잠재적인 공격 코드를 실행 시킬 수 있는 취약점입니다.

7) OpenSSL : BIO 통해 읽어 들인 ASN.1 데이터 처리에서의 서비스 거부 공격 (CVE-2016-2109) - OpenSSL을 사용하는 어플리케이션에 대하여 공격자가 OpenSSL 내부 자료구조(BIO)에 Overflow를 유도, 서비스 거부 공격이 가능한 취약점입니다.
