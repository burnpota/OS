안녕하세요,

현 이슈는 고객(박우진 선임)이 삼성생명 TAM(한진구 차장님)에게 직접 e-mail을 통해 다시 문의를 하여 진행중입니다.
CASE 진행사항을 보시면, 현재 아래와 같이 apache LogFormat에 \"X-Purpose: %{X-Purpose}i\" 을 추가하여 테스트 요청을 한 상태입니다.
참고하시기 바랍니다.

~~~~~
Apache Custom Log Formats [1]을 참조하면,
request header에서 foobar의 값을 출력하고 싶으면 %{foobar}i, response header에서 foobar의 값을 출력하고 싶으면 %{foobar}o 를 access log format에 설정할 수 있습니다.

따라서 현재 request header에 'X-Purpose: preview'이 존재한다면 httpd.conf의 LogFormat에 \"X-Purpose: %{X-Purpose}i\" 을 추가해볼 수 있겠습니다.

ex)
LogFormat "%h %l %u %t %T \"%r\" %>s %b \"%{Referer}i\" \"X-Purpose: %{X-Purpose}i\" \"%{User-Agent}i\" \"%{Cookie}i\"" combined

로컬에서는 확인할 위 설정을 추가해도 확인이 되지 않아 고객사 환경에 적용 및 확인이 필요합니다.
테스트 후 피드백 부탁드립니다.

[1] http://httpd.apache.org/docs/2.2/en/mod/mod_log_config.html#formats
~~~~~

감사합니다.
