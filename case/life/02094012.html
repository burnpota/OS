1. PPM의 의미

chronyc man page에 Frequency 관련 설명에 PPM에 대한 내용이 함께 설명되어 있습니다.
즉, 1 PPM은 시스템 시간이 1초 빠를 때 실제 시간과 비교하여 1.000001 초가 빠른다는 것을 의미합니다. 쉽게 생각하시면 백만분의 1 인 10의 마이너스 6승 만큼의 크기 입니다.

Frequency

The ‘frequency’ is the rate by which the system’s clock would be wrong if chronyd was not correcting it. It is expressed in ppm (parts per million). For example, a value of 1 ppm would mean that when the system’s clock thinks it has advanced 1 second, it has actually advanced by 1.000001 seconds relative to true time.

2. tinker panic 0

/etc/ntp.conf 파일 가장 맨 위에 넣어주시면 됩니다. 가상 머신에만 적용하셔야 합니다.

3. ntpd -qg

-g 옵션과 -q 옵션은 고객 포털 문서 Why does NTP error messsage " ntpd[xxxx]: frequency error -512 PPM exceeds tolerance 500 PPM " displayed in /var/log/messages ?[1]에 상세하게 설명되어 있습니다.

-g
Normally, ntpd exits if the offset exceeds the sanity limit, which is 1000 s by default. If the sanity limit is set to zero, no sanity checking is performed and any offset is acceptable. This option overrides the limit and allows the time to be set to any value without restriction; however, this can happen only once. After that, ntpd will exit if the limit is exceeded.
-q
Exit the ntpd just after the first time the clock is set. This behavior mimics that of the ntpdate program, which is to be retired. The -g and -x options can be used with this option.

즉, -g 옵션은 1000초이상 offset이 벌어지면 ntpd가 종료하게 되어 있는데 한번만 이 제한값에 적용받지 않고 동작하게 한다는 옵션이고,
-q 옵션은 시간동기화를 하고 난 후 ntpd 프로세스를 바로 종료하는 옵션입니다.

tinker panic 0 에 관련한 상세 설명[2]을 참고하시기 바랍니다.

안내해드리는 문서들을 꼼꼼히 읽어보시면 내용을 잘 아실 수 있습니다.

ntpd와 ntpdate의 옵션은 차이가 있습니다.

ntpd
-b Enable the client to synchronize to broadcast servers.
ntpdate
-b Force the time to be stepped using the settimeofday() system
call, rather than slewed (default) using the adjtime() system
call. This option should be used when called from a startup
file at boot time.

따라서, ntpd에서는 -b 옵션을 적용하지 않으시는 것이 좋을 것 같습니다.

> 2. 추가를 하면 강제적으로 동기화를 한번 진행하게 되는것인가요?
>

동기화하는 시점의 시간 차이에 따라 다른 함수를 호출하게 되어 있는데 기본값은 adjtime() 함수를 호출합니다. -b 옵션을 지정하면 항상 settimeofday() 함수를 호출하게 됩니다.

시스템 부팅시 ntpdate로 동기화를 한 번하게 되고 그 후 ntpd 데몬이 시작하여 동기화를 유지하게 됩니다.

> 3. 추가후에 ntpd 데몬만 재시작해주면 되는건가요?

-b 옵션은 ntpdate에만 반영되기 때문에 차후에 시스템 재시작시 반영됩니다.

Time adjustments are made by ntpdate in one of two ways. If ntpdate determines the clock is in error more than 0.5 second it will simply step the time by calling
