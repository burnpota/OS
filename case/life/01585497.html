안녕하세요,
Red Hat Technical Account Manager 한진구 입니다.

16일, 17일에 공개된 CVE-2015-7547와 CVE-2015-5229 관련 안내입니다.
리눅스 시스템 전반적으로 사용하는 핵심 라이브러리의 취약점이며 심각도가 높은만큼 신속하게 대응하실 수 있도록 사전 안내드립니다.

* CVE : https://access.redhat.com/security/cve/cve-2015-7547
- 대상 버전 : Red Hat Enterprise Linux 6, 7 및 파생제품
- 관련 패키지 : glibc
- 취약점 내용 : glibc의 라이브러이중 하나인 libresolv의 DNS 쿼리를 수행하는 코드에 스택오버플로우(Stack overflow) 확인. libresolv는 호스트네임과 IP간에 변환에 사용되는 함수를 포함하여 nss_dns 모듈에서 DNS lookup을 위해 libresolv를 사용. 해당 취약점을 이용해 외부 공격자가 특별히 조작된 DNS 응답을 생성 및 전달하여 lookup을 수행한 libresolv 라이브러리를 Crash시키거나 잠재적으로 libresolv를 사용하는 프로그램의 사용자 권한(root 포함)에서 코드를 실행시킬 수 있음.


* CVE : https://access.redhat.com/security/cve/CVE-2015-5229
- 대상 버젼 : Red Hat Enterprise Linux 6, 7 및 파생제품
- 관련 패키지: glibc
- 취약점 내용 : glibc안에 있는 'calloc' 함수를 사용할때, non-zero가 아닌 메모리 영역을 return함에 따라 application이 비정상적인 hangs 또는 crash가 유발될 수 있는 현상


* 영 향 도

1) 심각도 등급 : 심각 (Critical)

2) 영향 조건
- ssh 로그인, mail 서버, curl등 동작중 DNS request를 수행하는 모든 프로그램이 잠재적 공격 대상


* 해소 방안
- glib 패키지를 다음 버전으로 업데이트

- Errata

1) CVE-2015-5229
RHEL7, https://rhn.redhat.com/errata/RHSA-2016-0176.html
RHEL6, https://rhn.redhat.com/errata/RHBA-2015-1465.html

2) CVE-2015-7547
RHEL7, https://rhn.redhat.com/errata/RHSA-2016-0176.html
RHEL6, https://rhn.redhat.com/errata/RHSA-2016-0175.html


* 버젼졀 패키지

1) RHEL7

glibc-utils-2.17-106.el7_2.4
nscd-2.17-106.el7_2.4


2) RHEL 6
glibc-2.12-1.166.el6_7.7
glibc-common-2.12-1.166.el6_7.7
glibc-headers-2.12-1.166.el6_7.7
glibc-static-2.12-1.166.el6_7.7
glibc-utils-2.12-1.166.el6_7.7
nscd-2.12-1.166.el6_7.7


다수의 시스템을 운영하는 경우, Downtime이 없어야 하는 미션 크리티컬 시스템을 운영하는 경우, 업데이트와 재부팅이 어려움이 있다는 것 알고 있습니다. 다만 심각한 보안 취약점이고 언론을 포함 널리 알려진 사안인만큼 적극적인 대응이 필요할 것으로 생각됩니다.
