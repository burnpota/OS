안녕하세요,

Red Hat Global Support Services를 이용해주셔서 감사합니다.


1. 당시 mode 0이 권고를 해주신 이유와
2. lacp의 로구성하였을때의 영향도와 해당 구성방식이 권고방식이 아닌 이유를 알고 싶습니다

당시 mode 0으로 권고드린 이유에 대해서는 현재 관련자료 등이 확인이 되지 않고 있습니다.
이미 파악하고 계실 내용이겠지만 본딩 모드에 대해 설명드리자면 mode 0 은 active-backup 방식이기 때문에
active 슬레이브 인터페이스가 down이 될 경우 fail-over 를 통해 백업 인터페이스를 계속 사용할수 있습니다.

LACP(mode 4) 는 해당 기능을 지원하는 스위치와 함께 사용해야 합니다. 또한, mode 4를 사용하면 하나의 stream을 기준으로
load balancing 을 진행하게 됩니다. 즉 [1G(eth0)+1G(eth1)=2G(bond1)] 으로 구성되어 있다 하더라도 single stream을
사용하는 트래픽은 하나의 슬레이브만 사용하게 됩니다. single stream을 사용하는 경우는 대표적으로
NFS, CIFS, ISCSI, iperf가 있습니다. 이 경우는 더 빠른 속도를 사용하시려면 Mode 4에서 설정하는 것이
아닌 빠른 인터페이스(슬래이브)와 빠른 네트워크 인프라가 필요합니다. 또한 시스템이 기본적으로 트래픽을 받는 경우라면
스위치 차원에서 설정을 하셔야 합니다.

아래는 참고자료입니다.

Why are all interfaces not used in bonding Mode 2 or Mode 4?
-https://access.redhat.com/solutions/666853

Why does a LACP bond send or receive traffic down only a single link?
-https://access.redhat.com/solutions/372813

네트워크 구성은 이와 같이 환경에 의존적이기 때문에 권고 방식은 따로 없는 점 양해 부탁드립니다.
