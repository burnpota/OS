## 환경 ##

##


jbcs 설치 후 ${JBCS_HOME}/sbin 경로의 apachectl을 이용한 구동시 하기의 에러가 발생합니다.
## 에러스택 ##
/APP/jbcs/jbcs2423/httpd/sbin/httpd: error while loading shared libraries: libsystemd.so.0: cannot open shared object file: No such file or directory
##

해당 로그스택 확인결과 하기의 솔루션을 확인하였습니다.
참고 URL : https://access.redhat.com/solutions/2962621
##
문제
The file /lib64/libsystemd.so.0 does not exist and its absence prevents JBCS HTTPD from starting.

진단 단계
ldd httpd | grep libsystemd
libsystemd.so.0 => not found
해결
The /lib64/libsystemd.so.0 file is provided by the systemd-libs package, which must have at least version 219 installed.
##

하지만 해당 RHEL7에는 하기와 같이 systemd-lib 관련 패키지가 설치된 것으로 확인되어 다른 가이드
요청을 위해 SR 문의드리니 확인요청드립니다.
##
rpm -qa | grep systemd
systemd-219-42.el7.x86_64
systemd-sysv-219-42.el7.x86_64
systemd-python-219-42.el7.x86_64
systemd-libs-219-42.el7.x86_64
