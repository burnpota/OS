vmcore 분석 결과 아래와 같은 정보를 확인하였습니다.
1. 3rd party 모듈인 hpsa, Red Castle 운용
2. splunkd 애플리케이션 동작시에 페이지 할당 에러(page allocation failure) 발생
3. 메모리 부족 현상으로 oom-killer 발동
4. 이후에 panic_on_oom 파라메터 설정에 의한 시스템 패닉

따라서 아래와 같은 조치를 시도해볼 수 있습니다.
1. 레드햇 hpsa 드라이버로 운용
2. 최소 커널 메모리 증가(90112*3) vm.min_free_kbytes=270336 (/etc/sysctl.conf)
3. splunk 애플리케이션 제조사에 메모리 릭 확인,
성능 향상을 위한 THP 비활성화(https://access.redhat.com/solutions/46111)
메모리 튜닝 권고,
물리적인 메모리 증설 고려
4. 커널 최신버전으로 업데이트

원인 해결 이후에 panic_on_oom 파라메터는 비활성화 권고.
