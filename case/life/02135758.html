> 해당 파일들이 무슨 역활을 하는지 알수 있을까요?
> 현재 실행되고 있는 프로세스에 따라서 내용이 달라지는건인가요?
> 어떤것은 그냥 디렉토리만이 존재합니다

네, 생성되는 파일들은 데몬의 특성, 필요에 의해 따라 다를 수 있습니다.
예를 들어 데몬 이중 기동 방지를 위해 실행시 pid 파일 작성 , 정지시 pid 파일 삭제가
구현되어 있는 경우가 있습니다.
현재 실행되고 있는 데몬을 정지시키면 pid 파일이 사라지는 것을 확인하실 수 있습니다.

====
예)

# systemctl status httpd
httpd.service - The Apache HTTP Server
Loaded: loaded (/usr/lib/systemd/system/httpd.service; disabled)
Active: inactive (dead)

# systemctl start httpd

# ls -l
total 8
-rw-r--r--. 1 root root 8 Jul 9 09:31 authdigest_shm.1312
drwx------. 2 apache apache 40 Jun 25 09:10 htcacheclean
-rw-r--r--. 1 root root 5 Jul 9 09:31 httpd.pid
<=== 데몬 시작시 pid 파일 생성됨

# cat httpd.pid
1312

# systemctl stop httpd

# ls -l
total 0
drwx------. 2 apache apache 40 Jun 25 09:10 htcacheclean
<=== 데몬 정지시 pid 파일 삭제됨
====
